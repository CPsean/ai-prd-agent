# PRDs - 正式PRD输出区

> 本文件夹用于存放正式的产品需求文档（PRD），是产品经理的核心工作产出。

---

## 文件夹用途

正式PRD区是产品经理的核心输出区域，支持：

- **正式PRD文档**：经过审核、可交付开发的需求文档
- **版本管理**：PRD的版本迭代和历史记录
- **模块组织**：按模块组织PRD文档
- **归档管理**：过期版本的归档

---

## 组织方式

### 按模块组织

推荐使用以下文件夹结构：

```
prds/
├── 用户模块/
│   ├── 用户中心PRD-v1.0.md
│   ├── 用户中心PRD-v1.1.md
│   └── 字段清单/
├── 订单模块/
│   ├── 订单系统PRD-v1.0.md
│   └── 字段清单/
├── 支付模块/
│   └── 支付功能PRD-v1.0.md
└── archive/                    # 归档
```

### 文件命名规范

```
[功能名称]-PRD-v[版本号].md

示例：
- 用户中心-PRD-v1.0.md
- 订单系统-PRD-v2.0.md
- 支付功能-PRD-v1.0.md
```

## 审核标准

PRD进入此目录前应满足以下标准：

### 需求完整性

- [ ] 需求背景清晰
- [ ] 功能范围明确
- [ ] 业务规则完整
- [ ] 边界条件覆盖

### 文档规范性

- [ ] 遵循PRD模板
- [ ] 格式统一规范
- [ ] 图表清晰完整
- [ ] 引用准确有效

### 可执行性

- [ ] 需求描述具体
- [ ] 验收标准明确
- [ ] 技术方案可行
- [ ] 排期合理

### 协作性

- [ ] 已完成需求评审
- [ ] 已与开发确认
- [ ] 已与测试确认
- [ ] 已与设计确认

## PRD文档结构

标准PRD应包含以下章节：

```markdown
# [产品名称] PRD

## 1. 文档信息
- 版本历史
- 修订记录

## 2. 产品概述
- 产品背景
- 产品目标
- 目标用户

## 3. 功能需求
- 功能列表
- 功能详述
- 业务规则

## 4. 非功能需求
- 性能要求
- 安全要求
- 兼容性要求

## 5. 数据需求
- 数据实体
- 字段清单

## 6. 界面需求
- 页面清单
- 交互说明

## 7. 验收标准

## 8. 附录
```

## 版本管理

### 版本迭代规则

| 变更类型 | 版本变化 | 示例 |
|----------|----------|------|
| 重大变更 | 主版本号+1 | v1.0 → v2.0 |
| 功能新增 | 次版本号+1 | v1.0 → v1.1 |
| 错误修正 | 修订号+1 | v1.0.0 → v1.0.1 |

### 版本记录模板

```markdown
## 修订历史

| 版本 | 日期 | 修订人 | 修订内容 |
|------|------|--------|----------|
| v1.0 | 2026-02-27 | 张三 | 初始版本 |
| v1.1 | 2026-03-01 | 张三 | 新增XX功能 |
```

```
    ↓ 完善、审核

    ↓ 最终确认
outputs/（对外交付）
```

## 归档规则

### 何时归档

满足以下条件之一时，将PRD移至 `archive/`：

1. **项目已上线**：功能已上线并稳定运行
2. **需求已变更**：原PRD被新版本替代
3. **项目已取消**：项目取消不再推进

### 归档方式

```
prds/用户模块/用户中心-PRD-v1.0.md
    ↓ 移动到
prds/archive/用户模块/用户中心-PRD-v1.0.md
```

---

## 注意事项

1. **审核后进入**：只有经过审核的PRD才能放入此目录
2. **版本管理**：保留历史版本，便于追溯
3. **及时更新**：需求变更时及时更新版本
4. **关联字段清单**：PRD应关联对应的字段清单
