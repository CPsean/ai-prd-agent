# 签署页引导步骤条文案规范

> ✅ 需求ID：**15191**（20260205迭代，已上线）— 归属于「签署页前端优化内容集合」

---

## 用户故事

> **As a** 签署页用户（企业经办人或个人签署方），
> **I want to** 在签署过程中看到清晰的进度提示和操作引导文案，
> **so that** 我能快速了解还有哪些操作未完成，并被引导至下一步，减少因不知道"该做什么"而产生的困惑或中断。

---

## 功能概述

本文档规范签署页顶部引导步骤条（PC端）和底部操作区（H5端）的所有文案内容与展示逻辑，覆盖：
- 整体进度提示文案（"你有X个位置需要操作"）
- 引导按钮文案（"开始填写"、"下一个"、"提交"等）
- 各场景特殊文案（自由签、指定印章ID、纯填写、填签一体等）

**适用页面**：填签一体页（填写页与签署页合并后的统一页面），PC 端和 H5 端均适用

**架构层级**：L3 前端签署插件层

---

## 核心规则：X 计数逻辑

X 代表当前页面**必填未填项数量**，不依赖历史操作，动态实时计算。

| 计入 X 的条件 | 不计入 X 的条件 |
|--------------|----------------|
| 必填控件 + 未填值（含清空） | 签署申明强制阅读（前置阻断，不展示） |
| 动态：用户清空已填必填控件 → X+1 | 预填默认值控件（无论可编辑/不可编辑） |
| 动态：自由签手动添加必填控件 → X+1 | 独立签署时间控件、二维码控件 |
| 动态：填写了必填控件的值 → X-1（格式有误也算填过） | 数字控件（设置了公式计算）、人民币大写控件、合同编号 |
| | 附件（必填）→ 附件单独处理，不计入 X |

> 清空所有控件后，状态回退至"未开始"，提示"你有X个位置需要操作"。

**整体排序优先级（控件处理顺序）**：
> 签署申明 → 强制阅读 → 控件计算（含审批） → 附件（必填） → 可提交

**控件内部排序**：按文档从上到下顺序。

## PC 端文案规范

### 整体提示文案（步骤条上方）

| 状态 | 文案 |
|------|------|
| 流程未开始（所有控件未处理） | `你有X个位置需要操作` |
| 流程部分完成 | `你有X个位置需要操作` |
| 当前文档已全完成，但整个流程未完成（存在多份文件） | `你有X个位置需要操作，请切换下一份文件` |
| 已填值但格式有误 | `请调整内容格式` |
| 所有文档操作完成，但指定附件尚未上传 | `请按照要求上传附件` |
| 所有操作已完成 | `已完成所有指定操作，可点击提交` |

#### 纯自由签场景

| 状态 | 文案 |
|------|------|
| 未完成（含发起方指定印章类型但未完全覆盖） | `您还有签名未落，请双击或拖拽印章至任意位置盖章` |
| 拖动了签章控件但未指定印章，按指定位置处理 | `你有X个位置需要操作`（与部分完成合并） |
| 自由签全部已达标 | `已完成所有指定操作，您可提交或继续拖动印章到文档中后再提交` |

#### 指定印章ID场景（自动签章）

| 状态 | 文案 |
|------|------|
| 发起方已指定印章ID，系统自动匹配 | `已按照发起方设置自动匹配所有印章，无需修改可直接提交` |

---

### PC 端按钮文案

| 状态 | 文案 |
|------|------|
| **初始状态** — 第一个控件为填写控件 | `开始填写` |
| **初始状态** — 第一个控件为审批控件 | `开始审批` |
| **初始状态** — 第一个控件为备注签署控件 | `开始备注` |
| **初始状态** — 第一个控件为签署控件 | `开始签署` |
| **部分完成** — 默认 | `下一个` |
| **部分完成** — 当前页面已完成 | `下一份`（自动跳转至下一页最上方未填控件，无需页码提示） |
| **仅自由签场景** | 无操作 |
| **有指定附件未上传** | `上传附件`（点击后高亮附件上传区域） |
| **全流程完成** | 无操作 |

## H5 端文案规范

### 底部区域结构

- **副按钮**：承载详情、表单填写等辅助操作
- **主按钮**：引导用户定位必填未填控件（效果与PC跳转一致）
- **主按钮下提示文字**：文案与PC"整体提示"一致

---

### H5 端主按钮文案

| 状态 | 文案 |
|------|------|
| **初始状态** — 第一个控件为填写控件 | `开始填写` |
| **初始状态** — 第一个控件为审批控件 | `开始审批` |
| **初始状态** — 第一个控件为备注签署控件 | `开始备注` |
| **初始状态** — 第一个控件为签署控件 | `开始签署` |
| **部分完成** — 下一个未填为填写控件 | `继续填写`（点击后拉起填写表单并定位） |
| **部分完成** — 下一个未填为签署控件 | `签署文件`（点击后拉起印章选择界面并定位） |
| **部分完成** — 下一个未填为备注签署区 | `填写备注` |
| **部分完成** — 下一个未填为审批控件（国际站） | `点击审批` |
| **仅自由签场景** | `任意位置盖章` |
| **发起方指定印章类型但未完全覆盖** | `任意位置盖章`（点击时默认展开缺少的类型印章列表） |
| **当前页完成，存在下一份文件** | `下一份`（自动跳转至下一页最上方未填控件） |
| **指定必填附件未提交** | `上传附件` |
| **控件格式填写有误** | `继续填写` |
| **全流程完成** | `提交`（无操作提示） |

---

### H5 端副按钮规范

| 类型 | 规则 |
|------|------|
| **常驻按钮** | `详情`（始终显示） |
| **动态** — 页面存在签署控件时 | `继续盖章` |
| **动态** — 页面存在填写控件时 | `表单填写` |
| **动态** — 页面存在备注签署区时 | `继续备注` |

---

### H5 端主按钮下方提示文字

| 状态 | 文案 |
|------|------|
| 当前文档未开始 | `待操作内容X项` |
| 当前文档部分完成 | `待操作内容X项` |
| 所有文档完成，但指定附件未上传 | 无文案 |
| 信封所有操作已完成 | 无文案 |

## 验收标准

- [ ] **AC-1 X计数正确性**：必填未填控件计入X，预填默认值、独立签署时间控件、二维码控件、公式计算数字控件、人民币大写控件、合同编号均不计入X；附件单独处理不计入X
- [ ] **AC-2 PC整体提示 — 状态机**：覆盖"未开始 / 部分完成 / 跨文件提示 / 格式错误 / 附件待传 / 全完成"六种状态，文案与规范一致
- [ ] **AC-3 PC按钮文案 — 初始状态**：第一个控件类型分别为填写/审批/备注/签署时，展示对应文案
- [ ] **AC-4 PC按钮文案 — 部分完成**：当前页未完成显示"下一个"；当前页完成显示"下一份"并自动跳转
- [ ] **AC-5 H5主按钮 — 动态文案**：部分完成状态下，按下一个未填控件类型动态切换文案（填写/签署/备注/审批）
- [ ] **AC-6 H5副按钮 — 动态显示**："详情"常驻；"继续盖章"/"表单填写"/"继续备注"按页面控件类型存在与否动态显示
- [ ] **AC-7 自由签特殊文案**：纯自由签未完成时 PC 展示"您还有签名未落…"；H5主按钮展示"任意位置盖章"；全达标后 PC 展示"已完成所有指定操作，您可提交或继续拖动…"
- [ ] **AC-8 指定印章ID场景**：发起方已设置指定印章ID时，PC 整体提示展示"已按照发起方设置自动匹配所有印章，无需修改可直接提交"
- [ ] **AC-9 X值实时更新**：清空已填必填控件 → X+1；填写必填控件（格式有误也算）→ X-1；全清空后状态回退至"未开始"
