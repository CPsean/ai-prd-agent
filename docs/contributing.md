# 命令变更规范（维护者指南）

> 本文件供维护者修改斜杠命令时参考，不属于日常运行时加载内容。

---

## 三件套规范

任何斜杠命令的新增或修改，必须同步更新以下三个文件：

| 文件 | 作用 | 不更新的后果 |
|------|------|-------------|
| `.claude/commands/xxx.md` | 命令行为定义 | AI 执行逻辑与预期不符 |
| `CLAUDE.md` 速查表 + 目录结构 + 工作流 | AI 和用户的认知对齐 | 参数格式、流程描述与实际行为矛盾 |
| `evals/commands/TC-xxx.md` | 行为验证基准 | 测试通过但验证的是错误行为 |

**三者不一致 = 变更未完成。** AI 在修改命令定义后，应主动检查并同步另外两个文件。

**特别说明**：以下情况也属于三件套变更范围：
- 修改 context 文件的读取时机（如从"全量预读"改为"按章节按需读取"）
- 新增或删除某命令对某 context 文件的读取
- 修改 rdd.md 的 frontmatter 字段结构

---

## 模板变更额外规范

向 `templates/` 目录下的任何 PRD 模板新增 AI 指引内容时（编写规则、使用说明、context 文件引用提示等），**必须**为该内容添加 `[AI-ONLY]` 前缀标记：

```
> **[AI-ONLY]** 原始内容
```

**不加标记的后果**：新增 AI 指引内容会通过 `/new-prd` 直接泄露到生成的 PRD 中，被交付给团队或客户。

**不需要标记的内容**：
- 面向 PRD 读者（非 AI）的说明，如 `> 详细变更历史见同目录 CHANGELOG.md`
- 模板中作为实际内容占位的 blockquote，如 story-card 中的 `> **As a** [用户角色]...`
