# Drafts - 草稿区

> 本文件夹用于存放AI生成的初稿、快速记录、试错迭代等非正式文档。

---

## 文件夹用途

草稿区是产品经理的"沙盒"，支持：

- **AI生成初稿**：AI助手生成的初步内容
- **快速记录**：会议记录、灵感记录、临时笔记
- **试错迭代**：方案探索、多版本对比
- **工作过程**：思考过程、分析草稿

---

## 组织方式

### 按日期+主题组织

推荐使用以下文件夹结构：

```
drafts/
├── 2026-02-27-用户模块需求/
│   ├── 需求分析草稿.md
│   ├── 方案探索.md
│   └── 会议记录.md
├── 2026-02-28-订单流程优化/
│   ├── 流程分析.md
│   └── 方案对比.md
└── archive/                    # 归档
```

### 文件夹命名规范

```
[日期]-[主题描述]

示例：
- 2026-02-27-用户模块需求
- 2026-02-28-订单流程优化
- 2026-03-01-支付功能设计
```

---

## 归档规则

### 何时归档

满足以下条件之一时，将内容移至 `archive/`：

1. **内容已正式化**：草稿内容已整理到 `prds/` 或 `outputs/`
2. **方案已确定**：探索阶段结束，确定最终方案
3. **内容已过期**：不再需要保留的草稿

### 归档方式

```
drafts/2026-02-27-用户模块需求/
    ↓ 移动到
drafts/archive/2026-02-27-用户模块需求/
```

---

## 使用建议

### 草稿阶段

1. **快速记录**：不要追求完美，先记录想法
2. **多版本探索**：尝试多个方案，对比优劣
3. **AI协作**：让AI生成初稿，再逐步优化

### 正式化流程

```
drafts/（草稿）
    ↓ 审核、完善
prds/（正式PRD）
    ↓ 最终确认
outputs/（对外交付）
```

---

## 与其他文件夹的关系

| 场景 | 操作 |
|------|------|
| 草稿成熟 | 移动到 `prds/` |
| 需要交付 | 整理到 `outputs/` |
| 内容过期 | 移动到 `archive/` |
| 需要分析 | 参考到 `analysis/` |

---

## 注意事项

1. **草稿是临时的**：不要长期依赖草稿，及时正式化
2. **定期整理**：每周整理草稿区，归档过期内容
3. **命名清晰**：使用日期和主题命名，便于查找
