# 命令：需求澄清器（JTBD框架）

用 JTBD（Jobs-to-be-Done）框架分析以下需求，识别真实问题并生成RDD卡片：

$ARGUMENTS

## 执行步骤

**1. 输入有效性判断**：

先判断输入是否有足够信息可以分析：
- **极度模糊**（如"优化用户体验"、"做个功能"等，无法判断问题域）：列出 2-3 个可能方向，请用户先确认方向，**不继续后续步骤**
- **有方向但不清晰**：继续执行 Step 2

**2. X-Y问题诊断**：判断用户提出的是"解决方案(Y)"还是"真实问题(X)"。如果是Y，找出背后的X。

**3. JTBD追问**（如果真实问题不明确，提1-2个高价值问题）：
   > 当 [情境] 时，用户想要 [动机]，以便 [期望结果]。你真正希望达成的结果是什么？

   等待用户回答后继续。

**4. 价值回路验证**：确认如何衡量需求成功（指标/验证方法）。如果缺少指标，主动提出建议。

**5. 输出 RDD 卡片**（需求定义文档）：

```markdown
# 📋 RDD（需求定义文档）

- **⚠️ 核心洞察（X vs Y）**：[初始请求 vs 真实问题的对比说明]
- **🎯 JTBD / 用户故事**：作为 [角色]，我希望 [行为]，以便 [业务/用户价值]。
- **✅ 验收标准**：
    1. [标准1]
    2. [标准2]
- **🚀 MVP方案**：[验证假设的最轻量解法]
- **📈 成功指标**：[上线后如何衡量成功]
```

## 适用场景

需求提出但问题/价值不清晰时使用（Phase 1 需求发现阶段）。

不适用于：已有PRD的代码生成、Bug修复、技术架构设计。
