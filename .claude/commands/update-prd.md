# 命令：更新 PRD

对当前 PRD 进行变更，并自动维护版本记录：

$ARGUMENTS

## 使用方式

在已有 PRD 的文件夹内运行此命令，或通过参数指定 PRD 路径/标题。

`$ARGUMENTS` 描述本次变更内容，例如：
- "在业务规则章节补充登录态超时逻辑"
- "将验收标准 AC-2 的描述改为…"
- "删除页面 B，将其功能合并到页面 A"

## 执行步骤

**1. 定位目标 PRD**
- 如 `$ARGUMENTS` 包含路径或标题，按此定位
- 否则读取当前目录下的 `prd.md`
- 读取 YAML frontmatter 获取当前版本号

**2. 归档当前版本**
- 将当前 `prd.md` 完整复制到 `archive/prd-v[当前版本].md`
- 例如当前 version 为 "1.1"，则存为 `archive/prd-v1.1.md`

**3. 执行变更**
- 根据 `$ARGUMENTS` 的描述，精确修改 `prd.md` 中对应的章节内容
- 只改需要改的部分，不重写整个文件

**4. 更新 YAML**
- `version`：小版本号 +0.1（如 1.1 → 1.2）；若本次是重大重构则升主版本（1.x → 2.0）
- `updated`：今天日期

**5. 追加 CHANGELOG**

在 `CHANGELOG.md` 顶部（最新记录在最前）插入：
```markdown
## v[新版本] — [今天日期]

**变更类型**：[内容补充 / 逻辑修改 / 结构调整 / 问题修复]

**变更内容**：[具体描述本次修改了什么]

**影响范围**：[受影响的章节、页面、字段或接口]
```

**6. 更新 `prds/_registry.md`**
- 将对应行的"最后更新"字段改为今天日期

**7. 输出确认**

告知用户：
- 旧版本已归档至 `archive/prd-v[旧版本].md`
- 新版本号
- 本次修改的章节摘要

## 注意

- 不要修改 `archive/` 下的任何文件
- 如果变更描述模糊，先向用户确认具体修改内容，再执行
