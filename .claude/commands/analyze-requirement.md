# 命令：需求分析（Phase 1）

对以下需求进行深度分析：

$ARGUMENTS

## 执行步骤

按顺序完成以下分析，每步输出对应内容：

**1. 需求收集**：识别需求来源方、核心干系人、优先级（P0/P1/P2）

**2. 背景理解**：分析业务现状、核心痛点、业务目���与KPI

**3. 用户分析**：定义用户画像、主要使用场景（至少2个）、需求层次（基础/期望/兴奋）

**4. 需求澄清**：列出所有模糊点和待确认事项，标注影响决策的��键问题

**5. 优先级判断**：用MoSCoW方法（Must/Should/Could/Won't）对功能点分类，并给出分期建议

**6. 风险识别**：列出技术风险、业务风险、外部依赖

## 输出要求

生成完整的需求分析报告，存至 `analysis/` 目录，文件名：`YYYY-MM-DD-[需求名称]-analysis.md`

报告必须包含：背景 → 用户分析 → 功能范围 → 风险 → 分期规划

完成后进入 Phase 2：运行 `/design-solution`
