# 命令：数据模型设计（DDD）

用领域驱动设计（DDD）原则，为以下业务场景生成企业级数据库Schema：

$ARGUMENTS

## 执行步骤

**1. 领域分析**：
   - 识别聚合根（Aggregate Root）和实体（Entity）
   - 定义实体间关系（1:1 / 1:N / N:M）

**2. Schema定义**（使用以下企业级标准）：
   - 主键：雪花ID（BigInt）
   - 审计字段：`created_at` / `updated_at` / `created_by` / `updated_by`
   - 软删除：`is_deleted`
   - 乐观锁：`version`
   - 类型规范：金额/财务字段必须用 `DECIMAL(p,s)`，禁止 `FLOAT`/`DOUBLE`；优先用具体长度 `VARCHAR(N)` 而非 `TEXT`

**3. 索引策略**：根据查询模式建议索引，优先覆盖外键和高频过滤字段

**4. 输出内容**：

   a. **ER图**（Mermaid语法）

   b. **Schema表格**：

   | 字段名 | 类型 | 长度 | 必填 | 默认值 | 索引 | 说明 |
   |--------|------|------|------|--------|------|------|

   c. **设计说明**：聚合根选择理由、索引策略说明

## 失败处理

如果业务背景过于模糊无法识别实体，停止并提问：
> "请提供具体的业务实体或核心字段，以便继续建模。"

## 适用场景

用户提到"数据库设计"、"表结构"、"ER图"、"数据建模"时触发。
