# 命令：PRD 摘要

为指定 PRD 生成简洁的对齐摘要：

$ARGUMENTS

## 使用场景

- 评审会议前快速过一遍需求要点
- 研发/设计开始工作前的需求确认
- 需求变更后的"现在我们在哪"对齐

## 执行步骤

**1. 定位并读取 PRD**
- 读取 `prds/_registry.md`，按 ID 或标题关键词匹配目标 PRD
- **未找到匹配项时**：明确告知用户"在注册表中未找到该 PRD"，并提示：
  - 确认标题拼写是否正确
  - 若需新建，运行 `/new-prd`
  - 不得捏造或猜测 PRD 内容
- 找到后读取 `prd.md`（及 `fields.md` 如果存在）
- 读取 `CHANGELOG.md` 的最近 3 条记录

**2. 生成摘要**

输出以下结构（严格控制长度，每项不超过 3 行）：

---

## [PRD 标题] 摘要

**基本信息**：ID `[ID]` | 版本 `[version]` | 状态 `[status]` | 最后更新 `[updated]`

**核心用户故事**：
> As a [角色]，I want to [行为]，so that [价值]。

**功能范围**（包含）：
- [要点1]
- [要点2]
- [要点3]

**不包含**：
- [明确排除的内容]

**关键业务规则**：
- BR-XX：[规则摘要]

**验收标准**（共 N 条）：
- [ ] AC-1：[摘要]
- [ ] AC-2：[摘要]

**近期变更**：
- v[版本]（[日期]）：[变更摘要]

**开放问题**（如有）：
- ⚠️ [问题描述]

---

**3. 附加提示**

如发现 PRD 中有未填写的模板占位符（如 `[待补充]`），在摘要末尾提示：
"📝 PRD 中存在未完成章节：[章节名]，请运行 `/update-prd` 补充。"
