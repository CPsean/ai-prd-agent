# 命令：生成页面规格卡

基于已确认的 PRD，提取页面对象、区块结构、状态定义与操作说明，输出结构化页面规格卡：

$ARGUMENTS

## 前置条件

**只接受正式区 PRD 作为输入**，不接受草稿。

## 执行步骤

**1. 定位 PRD**

- 从 `$ARGUMENTS` 提取标题或 ID
- 查 `prds/_registry.md` 定位 PRD 路径，读取 `prd.md`
- 未找到时：停止执行，提示：
  > "该 PRD 尚未移入正式区（prds/），无法生成页面规格卡。请先确认 PRD 后再运行此命令。"

**2. 识别页面范围**

读取 PRD 的"页面 & 交互说明"章节，列出所有页面/弹窗/抽屉，输出页面清单并确认：

> 识别到以下页面对象，请确认范围是否完整：
> - [页面1]：[一句话说明]
> - [页面2]：[一句话说明]
>
> 有遗漏或需要调整的，请告知；确认无误后我开始生成规格卡。

等待用户确认后继续。

**3. 逐页生成页面规格卡**

每个页面按以下结构输出：

```markdown
## [页面名称]

**页面类型**：独立页面 / 弹窗 / 抽屉 / 嵌入区块
**触发入口**：[从哪里进入此页面]

### 区块结构

| 区块名称 | 包含内容 | 备注 |
|---------|---------|------|
| [筛选区] | [字段列表] | |
| [列表区] | [列名与说明] | |
| [操作区] | [按钮列表] | |

### 状态定义

| 状态 | 触发条件 | 页面表现 |
|------|---------|---------|
| 空状态 | 无数据时 | 空态文案 + 引导操作 |
| 加载中 | 数据请求中 | Skeleton 占位 |
| 异常态 | 请求失败 | 错误提示 + 重试入口 |

### 操作说明

| 操作名称 | 入口位置 | 触发条件 | 成功反馈 | 失败反馈 |
|---------|---------|---------|---------|---------|

### 关键文案

| 位置 | 文案内容 |
|------|---------|
```

**4. 保存页面规格卡**

在页面规格卡顶部插入元数据头：

```markdown
---
source-prd-version: "[当前 PRD version 字段值]"
last-updated: [今天日期]
---
```

将完整内容保存至 `prds/[标题]/page-spec.md`，并更新 `prd.md` 的 YAML frontmatter：
- 新增 `has-page-spec: true`
- 新增 `page-spec-path: page-spec.md`

**5. 输出确认与引导**

> **页面规格卡已生成，保存至 `prds/[标题]/page-spec.md`，共覆盖 [N] 个页面对象。**
>
> 确认规格卡内容无误后，运行 `/generate-prototype [标题]` 生成可交互原型。

> 如需调整某个页面的规格，直接告知我，我更新对应区块后重新保存。

## 适用场景

PRD 确认发布后、生成原型前使用。

不适用于：纯逻辑/后端类需求（无页面变更）、草稿阶段 PRD。
