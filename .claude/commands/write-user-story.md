# 命令：撰写用户故事

为以下需求生成符合规范的用户故事：

$ARGUMENTS

参考规范：`.claude/skills/user-story/SKILL.md`

## 输出格式

严格使用以下结构，不得简化：

```markdown
### 用户故事 [编号]：

- **摘要：** [以用户价值为中心的简短标题，非功能描述]

#### 使用场景：
- **As a** [具体用户角色，避免泛用"用户"]
- **I want to** [用户采取的行动]
- **so that** [期望达成的业务/用户价值——不得只是重复行动]

#### 验收标准：
- **场景：** [人类可读的场景描述]
- **Given：** [前置条件]
- **and Given：** [追加前置条件，可多个]
- **When：** [触发动作——只能有一个]
- **Then：** [期望结果——只能有一个]
```

## 质量检查（输出前逐项验证）

- [ ] "As a" 是具体角色，不是泛用"用户"
- [ ] "so that" 说明的是动机/价值，不是重复行动
- [ ] When 只有一个（多个 When = 需要拆分故事）
- [ ] Then 只有一个（多个 Then = 需要拆分故事）
- [ ] 摘要以价值为中心，不是功能描述

## 拆分提示

如果需求包含多个独立动作或用户角色，输出多个用户故事，每个独立成卡。

## 与 PRD 的衔接

生成后可直接复制到 `prd.md` 的"用户故事"章节，或作为独立 Story Card 用 `/new-prd story-card` 创建。
