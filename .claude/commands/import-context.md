# 命令：导入上下文（Import & Classify）

将以下内容导入到产品上下文文件中：

$ARGUMENTS

## 执行步骤

**1. 内容类型解析（CTX-IMP-001）**

识别 PM 提供的内容格式和业务含义：

**格式识别**：
- 纯文字 → 继续
- Markdown 文件 → 继续
- PNG/JPG 截图 → 走截图库流程（步骤 4）
- 不支持的格式（PDF/Figma/Word/Excel）→ 拒绝：「当前支持文字、.md 文件、图片截图，[格式] 暂不支持」，停止执行

**语义解析**：判断内容属于哪类业务信息：

| 业务含义 | 归属文件 |
|---------|---------|
| 产品背景/定位 | context/product-background.md |
| 产品策略/原则 | context/product-strategy.md |
| 用户角色/画像 | context/user-persona.md |
| 业务术语 | context/business-glossary.md |
| 功能模块信息 | context/product-feature-map.md |
| 权限规则 | context/permission-model.md |
| 页面截图 | context/screenshots/[模块]/ |

**语义模糊时**：追问 1 个区分问题（如「这段内容主要描述的是产品背景、用户信息、还是功能规则？」），不强行分类。

**内容与现有文件有重叠时**：标注重叠部分，询问是补充还是替换。

**2. 分类建议生成（CTX-IMP-002）**

生成分类建议表，批量展示：

> | # | 内容摘要 | 建议归类文件 | 建议写入位置 |
> |---|---------|------------|------------|
> | 1 | [摘要] | [文件名] | [章节/字段] |

内容涉及多个 context 文件时，拆分为多条建议行。

无合适归属文件时，标注「暂无合适归类，建议 PM 指定或新建上下文文件」。

展示后输出：
> 以上分类建议，回复「确认」批量写入，或指出需要修改的条目，或回复「取消」放弃导入。

等待 PM 确认（**阻塞**）。

**3. PM 确认写入（CTX-IMP-003）**

根据 PM 回复：

- **「确认」**→ 批量写入所有建议条目：
  - 追加方式写入目标文件对应章节，不覆盖已有内容
  - 写入前检查重复：相同信息已存在时提示跳过
  - 写入完成后输出摘要：「已写入 N 条内容：[目标文件列表]」

- **指出修改** → 更新该条分类，重新展示完整建议表，再次等待确认

- **「取消」**→ 不写入任何内容，结束流程

**目标文件不存在时**：询问 PM 是否创建该文件，确认后创建并写入。

**强制规则**：AI 不得在未获得 PM 确认的情况下写入任何 context 文件。

**4. 截图库流程（CTX-SCR-001 + CTX-SCR-002）**

仅当内容为截图时执行：

**4a. 功能模块归类存放（CTX-SCR-001）**

1. AI 解析截图内容，识别所属功能模块
2. 若无法自动识别：询问 PM「这个截图属于哪个功能模块？」
3. PM 确认模块后，AI 建议文件名（格式：[页面名称].png）
4. 写入 `context/screenshots/[功能模块]/[页面名].png`
5. 模块目录不存在时自动创建
6. 文件名重复时询问是否覆盖，或建议追加日期后缀

**4b. 页面跳转关系维护（CTX-SCR-002）**

截图存放后，询问跳转关系：

> [页面名] 是从哪个页面跳转过来的？跳转触发方式是什么？（如：点击[按钮名]）
> 回复跳转信息，或回复「跳过」。

- PM 提供跳转信息 → 更新 `context/screenshots/page-navigation.md`，在 Mermaid graph 中追加：
  ```
  来源页面 -->|点击[按钮]| 新页面
  ```
- PM 回复「跳过」→ 新页面节点独立存在，标注「待补充跳转关系」
- page-navigation.md 不存在时创建文件，以新页面为首个节点初始化 Mermaid 图
- Mermaid 节点 ID 不含空格，使用下划线替代
- 展示更新后的局部 Mermaid 图，确认是否正确
