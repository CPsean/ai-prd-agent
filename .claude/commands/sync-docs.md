# 命令：检查文档一致性

检查指定需求的 PRD、页面规格卡、原型是否保持一致，并检测飞轮待处理项，输出两块结构化报告：

$ARGUMENTS

## 执行步骤

**1. 文档存在性检测（SYNC-DET-001）**

- 从 `$ARGUMENTS` 提取标题或 ID
- 按优先级检查各文档是否存在：

| 文档 | 检查路径（正式区优先，草稿区备选） |
|------|--------------------------------|
| PRD | `prds/[标题]/prd.md` → `drafts/[标题]/prd.md` |
| 页面规格卡 | `prds/[标题]/page-spec.md` → `drafts/[标题]/page-spec.md` |
| 原型 | `outputs/prototypes/[标题]/index.html` |

- **PRD 不存在**：停止执行，提示"未找到「[标题]」的 PRD，请确认标题是否正确"。若检测到相似标题文件夹，提示供 PM 确认。
- **PRD 存在**：输出文档清单（✅ 存在 / ❌ 不存在），确定本次对比维度：
  - PRD + 规格卡 → PRD↔规格卡对比
  - PRD + 原型 → PRD↔原型对比
  - 规格卡 + 原型 → 规格卡↔原型对比
  - 仅 PRD → 跳过文档对比，输出"当前仅存在 PRD，无可对比文档"

**2. 文档内容对比（SYNC-CMP-001）**

按存在的文档组合执行内容级对比（非仅版本号对比）：

**PRD↔规格卡对比**（若两者均存在）：
- 页面清单：PRD 中描述的页面是否都在规格卡中有对应卡片
- 字段描述：规格卡字段与 PRD 数据字典是否一致
- 交互说明：规格卡交互描述与 PRD 功能说明是否对齐

**规格卡↔原型对比**（若两者均存在）：
- 页面数量与名称：规格卡中的页面是否在原型中都有对应 HTML
- 组件名称：规格卡关键组件是否在原型中体现

**PRD↔原型对比**（若两者均存在）：
- 页面覆盖度：PRD 中提及的页面原型是否都已生成

差异分类标注：
- **确定差异**：内容明确不一致（如规格卡新增了页面但 PRD 未提及）
- **疑似差异（请 PM 确认）**：语义相近但措辞不同，列出两处原文供 PM 比较

**2.5 Context 一致性检查**（context 文件存在时执行）

**术语孤岛检查**（`context/business-glossary.md` 存在时）：
- 读取 PRD §4 新引入术语，与 glossary 比对，找出未登记的术语

**功能前缀检查**（`context/product-feature-map.md` 存在时）：
- 读取 PRD §7 功能编号前缀，与前缀映射表比对，找出未注册的前缀

Context 文件均不存在时静默跳过，不输出此小节。

**3. 飞轮待处理检测（SYNC-FLY-001）**

检查 `context/pending-flywheel.md`：
- **存在且有未处理项**：列出所有待确认提议（含来源命令、提议日期）
- **存在但已全部处理**：输出"飞轮待处理项：已全部处理 ✅"
- **不存在**：输出"飞轮待处理项：无积压（文件不存在）"
- **格式异常**：提示"pending-flywheel.md 格式异常，请手动检查"，不中断检测

**4. 输出检测报告（SYNC-RPT-001）**

按固定两块格式输出：

```
## 文档同步检测报告 — [标题]

### 块一：文档一致性

检测范围：PRD ✅ / 页面规格卡 ✅/❌ / 原型 ✅/❌

**[维度名]**
- [差异项1]（确定差异）：[定位] → [问题描述]
- [差异项2]（疑似差异，请确认）：[PRD原文] vs [规格卡原文]

> 无差异时：✅ 文档内容一致，无差异

**Context 一致性：**（仅 context 文件存在时输出）
- 术语孤岛：[列表]
- 功能前缀未注册：[列表]
> 均无问题时：✅ Context 一致性：无孤岛

### 块二：飞轮待处理项

- [提议1]（来源：/requirement-clarifier，2026-04-18）
- [提议2]（来源：/new-prd，2026-04-20）

> 处理方式：回复"确认[提议]"写入对应 context 文件，或回复"跳过"标记为已忽略
> 无待处理时：✅ 飞轮待处理项已全部处理
```

所有维度均无差异且飞轮已清零时，输出：`✅ 文档健康，无需处理`

**5. 询问是否立即执行**

> 是否现在开始同步？请告知需要处理哪几条，或回复"全部"由 AI 依次执行。

仅在有差异或待处理项时询问。全部健康时不询问。

## 适用场景

- PRD 更新后，检查 page-spec 和原型是否需要同步
- 原型评审后发现问题，反向检查 PRD 是否需要更新
- 阶段性做全面文档核查
- 清零飞轮积压的上下文更新提议

不适用于：草稿区 PRD 尚无任何关联文档的场景（仅做飞轮检测）。
