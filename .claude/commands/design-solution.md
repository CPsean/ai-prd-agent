# 命令：方案架构（Phase 2）

基于需求分析报告，设计完整的产品方案：

$ARGUMENTS

**前置**：先读取 `analysis/` 目录下最新的需求分析报告作为输入上下文。

## 执行步骤

**1. 设计目标确认**：提炼核心设计目标（≤3条）、约束条件（时间/资源/技术/业务）

**2. 信息架构**：输出产品模块树形结构（用缩进层级表示），定义主导航和页面清单

**3. 功能流程**：
   - 核心业务流程（用有序步骤描述，复杂流程用 Mermaid flowchart）
   - 状态流转表（状态 → 触发条件 → 转换后状态）
   - 异常处理方案

**4. 交互方案**：关键页面的布局描述、核心交互规则、错误提示规范

**5. 数据模型**：定义核心数据实体和关系，输出字段清单框架（详细字段在 Phase 3 生成）

**6. 接口规划**：列出接口清单（名称 / 方法 / 路径 / 用途）

**7. 方案自查**：完整性、技术可行性、用户体验、安全性逐项评估

## 输出要求

生成方案设计文档，存至 `drafts/` 目录，文件名：`YYYY-MM-DD-[需求名称]-solution.md`

完成后进入 Phase 3：运行 `/generate-specs`
