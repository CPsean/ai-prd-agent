# 命令：PRD 问答

基于 PRD 内容，回答关于需求的具体问题：

$ARGUMENTS

## 使用说明

任何人（产品、研发、测试、设计）都可以用此命令提问，例如：
- `/prd-qa 用户登录模块的 Session 超时时间是多少？`
- `/prd-qa F-001 的验收标准第二条具体是什么？`
- `/prd-qa 这个功能的字段清单在哪里？`

## 执行步骤

**1. 定位目标 PRD**

按以下优先级查找：
- `$ARGUMENTS` 中包含 PRD ID（如 F-001）→ 查 `prds/_registry.md` 找到路径，读取对应 `prd.md`
- `$ARGUMENTS` 中包含功能名称关键词 → 查 `prds/_registry.md` 匹配标题，读取对应 `prd.md`
- 未指定 → 询问用户"请问是关于哪个功能的问题？"

**2. 只读当前版本**
- 只读 `prd.md`，不读 `archive/` 下的历史文件
- 如有 `fields.md`，一并读取

**3. 回答问题**

回答格式要求：
- 直接给出答案，不要大段复述 PRD 原文
- 如果 PRD 中没有明确定义，说明"PRD 中未定义此项"，并建议运行 `/update-prd` 补充
- 如果问题涉及多个章节，注明来源章节（如"来自业务规则 BR-02"）
- 引用原文时使用引号

**4. 发现 PRD 缺陷时主动提示**

回答问题的同时，如果发现：
- 问到的内容 PRD 中没有定义
- 存在明显的逻辑矛盾
- 某个边界条件没有覆盖

主动提示："⚠️ 发现潜在缺陷：[描述]，建议运行 `/update-prd` 补充。"
