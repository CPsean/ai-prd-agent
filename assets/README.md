# Assets - 资源文件库

> 本文件夹用于统一管理项目中的图片、图标、原型图、流程图等资源文件。

---

## 文件夹用途

资源文件库是项目中所有静态资源的集中管理区域，支持：

- **图片素材**：产品截图、示例图片、宣传图等
- **图标**：功能图标、状态图标、品牌图标等
- **原型图/设计稿**：UI设计稿、交互原型、视觉稿等
- **流程图/架构图**：业务流程图、系统架构图、数据流图等

---

## 子文件夹说明

### images/

存放图片素材：
- 产品截图
- 示例图片
- 宣传图片

存放图标资源：
- 功能图标
- 状态图标
- 品牌图标
- 装饰图标

### mockups/

存放原型图和设计稿：
- UI设计稿
- 交互原型
- 视觉稿
- 设计规范

### diagrams/

存放流程图和架构图源文件：
- 业务流程图
- 系统架构图
- 数据流图
- 状态机图

## 资源命名规范

推荐使用以下命名格式：

```
[模块名]-[内容描述]-[版本号].[扩展名]

示例：
- user-login-page-v1.0.png
- order-flow-diagram-v1.2.drawio
- dashboard-mockup-v2.0.fig
```

**命名规则**：
- 使用英文小写
- 单词间用连字符 `-` 分隔
- 版本号格式：`v1.0`、`v1.1`、`v2.0`
- 文件名清晰描述内容

---

## 引用方式

在Markdown文档中引用资源：

```markdown
![图片描述](../assets/images/user-login-page-v1.0.png)
```

在PRD文档中引用原型图：

```markdown
## 界面原型

详见：[登录页面原型](../assets/mockups/user-login-page-v1.0.png)
```

---

## 使用建议

1. **统一存放**：所有资源文件统一存放在此目录
2. **规范命名**：遵循命名规范，便于查找和维护
3. **版本管理**：重要资源保留历史版本
4. **定期清理**：清理不再使用的资源文件

---

## 支持的文件格式

| 类型 | 支持格式 |
|------|----------|
| 图片 | `.png`, `.jpg`, `.jpeg`, `.gif`, `.svg`, `.webp` |
| 图标 | `.svg`, `.png`, `.ico` |
| 设计稿 | `.fig`, `.sketch`, `.xd` |
| 流程图 | `.drawio`, `.vsd`, `.vsdx` |
