# 迭代需求列表

> **用途**：供 `/ingest-prd` 在录入历史 PRD 时自动匹配需求 ID、所属迭代、优先级和负责人。
> **维护方式**：每次迭代规划后，将新增需求追加到表格中。
> **非必填**：不使用历史 PRD 录入功能时，此文件可为空。

---

## 初始化说明

请在首次使用前填写以下需求清单。完成后删除本段说明。

**需要填写的内容：**
1. 历史需求的 ID、标题、所属迭代
2. 计划上线日期和优先级
3. 负责人和当前状态

## 使用说明

- **需求ID**：与 `prds/_registry.md` 中的 ID 保持一致（SC-XXX / F-XXX / E-XXX）
- **所属迭代**：建议用季度（2026-Q1）或迭代编号（Sprint-12）
- **优先级**：P0（必做）/ P1（重要）/ P2（按需）
- **状态**：规划中 / 开发中 / 已上线 / 已废弃
