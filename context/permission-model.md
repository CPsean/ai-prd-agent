## 权限模型类型

> 请选择并保留适用的一项，删除其余：

- **RBAC**（基于角色）：权限绑定在角色上，用户通过角色获得权限。适合层级清晰、角色固定的系统。
- **ABAC**（基于属性）：权限由主体、资源、环境的属性组合决定。适合细粒度动态权限场景。
- **PBAC**（基于策略）：通过策略规则引擎评估权限，兼容 RBAC 和 ABAC。适合复杂合规场景。

## 权限粒度规则

> 说明权限控制的粒度层级，方便 PRD 中描述"某角色只能操作某范围内的数据"。

**操作权限粒度**：
- （示例）读 / 写 / 删除 / 导出 / 审批

**数据范围粒度**：
- （示例）全局 / 租户级 / 部门级 / 个人级

**特殊规则**：
- （示例）超级管理员不受租户隔离限制
- （示例）被授权人可在授权范围内操作他人数据
