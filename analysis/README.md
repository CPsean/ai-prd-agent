# Analysis - 业务分析区

> 本文件夹用于存放业务分析相关文档，包括数据分析、流程推演和影响范围分析。

---

## 文件夹用途

业务分析区是产品经理进行深度分析的工作区域，支持：

- **数据分析**：业务数据分析、用户行为分析、指标分析
- **流程推演**：业务流程模拟、场景推演、边界条件分析
- **影响范围分析**：功能变更影响、系统依赖分析、风险评估

---

## 子文件夹说明

### data-analysis/

存放数据分析相关文档：
- 业务数据分析报告
- 用户行为分析
- 指标分析报告
- 数据可视化图表

### process-simulation/

存放流程推演相关文档：
- 业务流程图
- 场景推演文档
- 边界条件分析
- 异常流程处理

### scope-analysis/

存放影响范围分析相关文档：
- 功能变更影响分析
- 系统依赖分析
- 风险评估报告
- 兼容性分析

---

## 文件命名规范

推荐使用以下命名格式：

```
[类型]-[模块名]-[内容描述]-[日期].md

示例：
- analysis-用户模块-用户行为分析-20260227.md
- process-订单流程-异常场景推演-20260227.md
- scope-支付功能-变更影响分析-20260227.md
```

**类型缩写**：
- `analysis` - 数据分析
- `process` - 流程推演
- `scope` - 影响范围分析

---

## 使用建议

1. **分析前准备**：明确分析目标和关键问题
2. **数据收集**：在 `docs/01-reference/` 中存放原始数据
3. **分析过程**：使用结构化的分析方法
4. **结论输出**：形成可执行的建议
