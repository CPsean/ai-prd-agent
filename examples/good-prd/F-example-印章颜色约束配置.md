# 指定个人签名印章的模板章及笔迹签名颜色约束

## 用户故事

**主故事**
> **As a** 租户管理员，
> **I want to** 在流程配置中统一约定个人和上下方可使用的印章颜色范围（模板章颜色、笔迹签名颜色），
> **so that** 确保所有签署人的印章符合企业规范，发起方无需逐一管控。

**补充故事**
> **As a** 签署人，
> **I want to** 在签署页只看到符合发起方颜色要求的印章，
> **so that** 我可以直接完成签署，不会因印章颜色不符导致流程中断。

---

## 功能概述

当前系统中印章颜色由用户自行决定，发起方无法强制要求个人和上下方及最终签名人使用特定颜色的印章。

本功能通过**租户级颜色偏好配置**，使发起方能够统一约定模板章颜色（red/black/blue/purple）和笔迹签名颜色（red/black）。签署页加载时读取配置并过滤印章列表，若无可用印章则引导用户按要求创建。配置变更仅对后续**新发起的流程**生效；存量流程不受影响。

## 页面 & 交互说明

### 页面 A：后台管理端 — 印章颜色偏好设置

**页面用途**：租户管理员统一配置印章颜色约束范围。

**页面元素**：
- 模板章颜色（多选）：red、black、blue、purple，开启后默认勾选 red
- 笔迹签名颜色（多选）：red、black，开启后默认勾选 red
- 平台级配置（7488）作为底层默认值，租户配置优先级高于平台配置

**交互规则**：
- 管理员保存配置后，仅对**新发起的流程**生效
- 未配置时，签署页不做颜色过滤，兼容现有行为

---

### 页面 B：签署页 — 印章列表（已有页面改造）

**页面用途**：签署人在签署流程中选择印章。

**交互规则**：
- 页面加载时请求接口获取当前租户的颜色约束配置
- 过滤当前用户印章列表，仅显示颜色属性符合配置要求的印章
- **系统不默认选中任何印章**，由用户手动点击选择
- 过滤后列表为空时，显示"去创建"入口

**空状态处理**：
- 过滤后无可用印章 → 显示"去创建"按钮，点击后新标签页打开印章创建页
- 返回签署页显示蒙层弹窗：「你正在进行创建印章操作流程，完成后印章会显示在列表中，请确认后刷新」
- 弹窗提供"刷新页面"按钮，点击后刷新整个页面重新加载印章列表

---

### 页面 C：印章创建页（URL 参数改造）

**页面用途**：用户创建符合颜色要求的新印章。

**交互规则**：
- 通过 URL 参数传入颜色约束
- 创建页中颜色选择控件被锁定，仅允许选择发起方指定范围内的颜色

---

## 业务规则

| 规则编号 | 规则描述 | 备注 |
|----------|----------|------|
| BR-01 | 颜色配置优先级：租户级配置 > 平台级 7488 配置，租户未配置时以 7488 配置为默认约束 | 平台配置作为 fallback，限定而不限死 |
| BR-02 | 颜色配置变更仅对变更后新发起的流程生效，存量流程保持发起时的颜色配置 | 新旧流程隔离 |
| BR-03 | 首次进入签署页且无印章时，自动生成规则：配置范围包含 red 则生成红色模板章，否则随机生成一枚符合要求颜色的 | 自动生成兜底 |
| BR-04 | 本次发起方不开放按个人流程维度的颜色覆盖，只读取租户级配置 | 本次范围限制 |
| BR-05 | 以上颜色约束逻辑同样适用于**企业结合人印章** | 结合人覆盖 |

---

## 边界条件 & 异常处理

| 场景 | 处理方式 |
|------|----------|
| 租户未配置颜色偏好 | 回退使用平台级 7488 配置作为颜色约束，而非完全不限制 |
| 配置修改后，旧流程中的签署任务 | 继续使用发起时的颜色配置，不受影响 |
| 用户完成创建印章后返回签署页 | 需手动点击"刷新页面"，系统不自动轮询 |
| 新创建的印章不符合颜色要求 | 创建页已通过 URL 参数锁定颜色范围，理论上不会出现，若出现仅通过过滤处理 |

## 字段清单

### 模块 1：租户颜色偏好配置（管理端写入）

| 字段名 | 标识符 | 类型 | 必填 | 枚举值 / 说明 | 业务规则 |
|--------|--------|------|------|--------------|----------|
| 模板章允许颜色 | `template_seal_colors` | array[enum] | 否 | red, black, blue, purple | 空数组 = 不限制，沿用平台 7488 默认约束 |
| 笔迹签名允许颜色 | `handwriting_colors` | array[enum] | 否 | red, black | 空数组 = 不限制，沿用平台 7488 默认约束 |

### 模块 2：颜色约束读取（签署页接口响应）

| 字段名 | 标识符 | 类型 | 必填 | 说明 | 业务规则 |
|--------|--------|------|------|------|----------|
| 生效模板章颜色范围 | `effective_template_colors` | array[enum] | 是 | 合并租户配置与平台 7488 默认后的生效约束 | 租户有配置则用租户的，否则用平台的 |
| 生效笔迹签名颜色范围 | `effective_handwriting_colors` | array[enum] | 是 | 同上 | 同上 |
| 配置来源 | `config_source` | enum | 是 | tenant_config / platform_default | 用于前端判断是否展示"由租户配置"提示 |

## 非功能需求

| 类型 | 要求 |
|------|------|
| 性能 | 签署页加载颜色约束配置接口响应时间 ≤ 200ms（P95） |
| 安全 | 颜色约束通过接口下发，创建页通过 URL 参数锁定，需防止参数篡改（服务端二次校验） |
| 兼容性 | 配置变更不影响已发起流程中的签署行为 |

## 验收标准

- [ ] **AC-1 配置生效校验**：管理员设置仅允许 blue，签署人进入页面后，red/black 印章不出现在可选列表中
- [ ] **AC-2 新旧流程兼容**：流程 A 发起后管理员修改配置，流程 A 的签署过程仍保持发起时的颜色要求，新发起的流程 B 遵循新配置
- [ ] **AC-3 创建闭环校验**：签署人点击"去创建"，在弹出页中只能选择符合要求的颜色，创建成功返回签署页点击刷新后，新创建的印章出现在列表中
- [ ] **AC-4 自动生成校验**：首次签署用户进入时，系统自动生成至少一枚符合颜色要求的模板印章（含 red 时优先生成红色，否则随机生成一枚）
- [ ] **AC-5 企业结合人印章**：以上流程在结合人的章场景中同样生效
