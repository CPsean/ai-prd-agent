# 示例：质检报告输出样本

> **说明**：此为 `/prd-summary` 和 `/update-prd` 末尾质检步骤的预期输出格式，包含通过、待完善、未通过三种情况的样本。

---

## 样本 A：质量良好

```
## PRD 质检报告 — 消息通知设置（SC-000）

✅ 通过（18项）
⚠️ 待完善（0项）
❌ 未通过（0项）

本 PRD 质量良好，可进入开发阶段。
```

---

## 样本 B：有待完善项（不阻断）

```
## PRD 质检报告 — 订单列表筛选（F-003）

✅ 通过（14项）
⚠️ 待完善（3项）
❌ 未通过（0项）

⚠️ 待完善项：

G3-04  验收标准未覆盖异常场景（当前只有正常流程的 AC）
       建议：补充接口失败、空列表、网络超时场景的验收标准

G5-03  字段清单中"筛选条件"字段未出现在页面说明中，或页面中提到的"排序字段"未在清单中定义
       建议：对照页面说明检查字段清单完整性

G6-01  开放问题 Q2 缺少负责方
       建议：补充"负责确认人：[姓名]"

以上为建议项，不影响 PRD 状态推进，但建议在开发启动前补充完整。
```

---

## 样本 C：有必须修复项（阻断）

```
## PRD 质检报告 — 支付结果页（F-007）

✅ 通过（10项）
⚠️ 待完善（1项）
❌ 未通过（3项）

❌ 必须修复（阻断推进）：

G2-01  缺少用户故事章节
       当前状态：PRD 正文直接从"功能概述"开始，未包含用户故事
       修复方式：补充符合 As a / I want to / So that 格式的主用户故事

G3-02  验收标准 AC-2 无法测试：「支付体验顺畅」
       当前写法：AC-2：支付体验顺畅，无卡顿
       修复建议：AC-2：支付结果页从接口返回到页面展示完成 ≤ 1 秒（P95，4G 网络）

G4-01  缺少边界条件 & 异常处理章节
       当前状态：文档中未出现该章节
       修复方式：补充至少以下场景：支付超时、支付失败、重复支付、网络断开

⚠️ 待完善项：

G5-01  字段清单为空占位符，建议补充支付结果核心字段

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
以上 3 项❌为必须修复项，请修复后再运行 /prd-summary 或推进状态。
```
