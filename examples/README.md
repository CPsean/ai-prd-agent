# examples/ — 示例库

> **AI 使用说明**：生成 PRD 内容或 skill 输出时，优先参考此目录中的示例对齐输出质量和风格。示例是"对齐输出"的，不是"模板结构"——结构在 `templates/` 中，这里的示例展示的是填充后的预期效果。

---

## 子目录说明

| 目录 | 内容 | AI 何时参考 |
|------|------|------------|
| `good-prd/` | 高质量 PRD 完整案例 | 生成或审查 PRD 内容时，对齐表达精度和完整度 |
| `anti-patterns/` | 典型反面案例 + 批注 | 质检时识别模糊表达、不完整结构 |
| `skill-outputs/` | 命令输出的预期样本 | 执行 skill 时，对齐输出格式和信息密度 |

---

## 使用边界

- 示例仅作参考，不可直接复制内容到真实 PRD
- 示例中的业务场景为通用虚构场景，与实际产品无关
- 优先使用 `good-prd/` 中的风格，遇到质量问题时对照 `anti-patterns/` 识别根因

---

## 维护说明

- 积累到足够真实 PRD 后，将典型案例（脱敏后）移入 `good-prd/`
- 发现新的反面模式时，追加到 `anti-patterns/`
- skill 输出格式变更后，同步更新 `skill-outputs/` 中的样本
