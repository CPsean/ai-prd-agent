# 测试用例：/sync-docs 命令

> 关联命令：`.claude/commands/sync-docs.md`
> 参数格式：`/sync-docs [标题或ID]`
> 前置条件：目标 PRD 已在 `prds/` 正式区注册，存在关联文档（page-spec / 原型）

---

## TC-SD-01 发现 page-spec 落后于当前 PRD 版本

| 字段 | 内容 |
|------|------|
| **状态** | ✅ 2026-04-10 |
| **测试目标** | 验证 PRD 更新后 page-spec 未同步时，AI 正确识别差异并列出建议操作 |
| **前置条件** | `prds/消息推送设置/prd.md` 为 v1.2，`page-spec.md` 的 `source-prd-version` 为 `1.0`，CHANGELOG 有 v1.1 和 v1.2 变更记录，其中 v1.1 涉及页面布局调整 |

**测试输入**
```
/sync-docs 消息推送设置
```

**预期行为**
1. 读取 `prd.md`（version=1.2）、`CHANGELOG.md`、`page-spec.md`（source-prd-version=1.0）
2. 识别 page-spec 基于 v1.0，PRD 已有 v1.1、v1.2 更新
3. 从 CHANGELOG 找出 v1.1 和 v1.2 的变更内容
4. 判断哪些变更影响页面（v1.1 涉及布局调整 → 影响 page-spec 和原型）
5. 输出检查报告：说明差异条目、影响范围、建议操作
6. 询问是否立即开始同步

**检查要点**
- [ ] 报告中显示"当前 PRD 版本：v1.2"
- [ ] 报告中显示"页面规格卡基于：PRD v1.0"
- [ ] 识别出至少 1 条涉及页面的变更（v1.1 布局调整）
- [ ] 每条差异条目包含：变更类型、变更内容、影响范围、建议操作
- [ ] 建议操作包含 `/generate-page-spec` 和 `/generate-prototype` 命令
- [ ] 末尾询问是否立即执行同步

## TC-SD-06 原型落后于 page-spec（page-spec 已更新但原型未重新生成）

| 字段 | 内容 |
|------|------|
| **状态** | ✅ 2026-04-10 |
| **测试目标** | 验证原型版本落后于 page-spec 时，AI 能识别并建议重新生成原型 |
| **前置条件** | `prd.md` v1.2，`page-spec.md` `source-prd-version: 1.2`，`prototype/index.html` `generated-from-prd-version: 1.0` |

**测试输入**
```
/sync-docs 消息推送设置
```

**预期行为**
- page-spec 与 PRD 一致（均为 v1.2）
- 原型基于旧版本 v1.0，落后于当前 page-spec
- 输出：识别到原型与 page-spec 版本不一致，建议重新生成原型

**检查要点**
- [ ] 报告中区分 page-spec 状态（一致）和原型状态（落后）
- [ ] 建议操作为 `/generate-prototype [标题]`
- [ ] **不**建议重新运行 `/generate-page-spec`（page-spec 已是最新）
