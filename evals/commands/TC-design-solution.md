# 测试用例：/design-solution 命令

> 关联命令：`.claude/commands/design-solution.md`
> 参数格式：`/design-solution`（无参数，自动读取最新分析报告）
> 前置条件：`analysis/` 目录下存在需求分析报告

---

## TC-DS-01 正常方案设计（基于已有分析报告）

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **测试目标** | 验证基于分析报告生成完整方案文档，包含所有必要章节 |
| **前置条件** | `analysis/` 下存在最新分析报告（执行 TC-AR-01 后） |

**测试输入**
```
/design-solution
```

**预期行为**
1. 自动读取 `analysis/` 目录下最新的分析报告
2. 提炼核心设计目标（≤3 条）
3. 输出信息架构（模块树形结构）
4. 描述核心业务流程（步骤或 Mermaid 流程图）
5. 定义状态流转表
6. 输出字段清单框架和接口规划
7. 执行方案自查
8. 生成文档至 `drafts/YYYY-MM-DD-[需求名称]-solution.md`

**检查要点**
- [ ] 方案文档存在于 `drafts/` 目录，文件名格式正确
- [ ] 包含设计目标、信息架构、功能流程、交互方案、数据模型、接口规划、方案自查七个章节
- [ ] 设计目标 ≤ 3 条
- [ ] 状态流转表格式正确（状态 → 触发条件 → 转换后状态）
- [ ] 方案自查覆盖完整性、可行性、体验、安全性
- [ ] 提示了后续操作（`/new-prd` 或 `/write-user-story`）

---

## TC-DS-02 无分析报告时的处理

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **测试目标** | 验证 analysis/ 目录为空时 AI 正确提示，不凭空设计 |
| **前置条件** | `analysis/` 目录为空或不存在 |

**测试输入**
```
/design-solution
```

**预期行为**
- AI 检查 `analysis/` 目录，发现无分析报告
- 告知用户需要先运行 `/analyze-requirement` 完成 Phase 1
- **不**凭空生成方案文档

**检查要点**
- [ ] 未创建任何方案文档
- [ ] 明确提示需先运行 `/analyze-requirement`
- [ ] 未捏造需求背景或方案内容

## TC-DS-03 方案中包含 Mermaid 流程图

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **测试目标** | 验证复杂流程时输出 Mermaid 流程图，格式正确可渲染 |
| **前置条件** | 分析报告中包含多步骤业务流程 |

**测试输入**
```
/design-solution
```
（基于包含审批流程等多步骤业务的分析报告）

**预期行为**
1. 识别到核心业务流程较复杂（多步骤、多分支）
2. 在功能流程章节使用 Mermaid flowchart 语法
3. 流程图包含正常流和异常流

**检查要点**
- [ ] 方案文档中存在 Mermaid 代码块（```mermaid）
- [ ] 流程图语法正确（flowchart TD/LR 格式）
- [ ] 包含判断节点和异常分支
