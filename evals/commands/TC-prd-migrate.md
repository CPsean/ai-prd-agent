# 测试用例：存量 PRD 迁移方案

> 关联 PRD：F-009 PRD生命周期一致性（PRD-MIG-005）
> 前置条件：`prds/_registry.md` 存在且有活跃 PRD

---

## TC-MIG-01 存量迁移清单输出

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **测试目标** | 验证用户请求存量迁移时，AI 输出完整的对照表、更新说明、清理清单和验收检查步骤 |
| **前置条件** | `prds/_registry.md` 中有多条活跃 PRD，`prds/` 下存在不含 ID 前缀的文件夹（如 `prds/需求澄清/`、`prds/PRD生成/`）；`drafts/` 中可能有孤立目录 |

**测试输入**
```
帮我迁移历史文件夹
```

**预期行为**
1. AI 读取 `prds/_registry.md` 所有活跃 PRD 行
2. 对比 `prds/` 实际目录结构，识别不含 ID 前缀的文件夹
3. 输出迁移清单，包含以下四部分：
   - **第一步：重命名 prds/ 文件夹**（原文件夹 → 新文件夹 → 对应 ID 对照表）
   - **第二步：更新 prds/_registry.md 路径列**（说明如何修改路径格式）
   - **第三步：清理 drafts/ 中的孤立目录**（列出 prds/ 已有对应 PRD 的 drafts/ 孤立目录）
   - **验收检查**（列出可运行的验证命令）
4. AI 不自动执行任何文件操作，仅输出清单

**检查要点**
- [ ] 输出包含"重命名 prds/ 文件夹"对照表，格式为「原文件夹 → 新文件夹 → 对应 ID」
- [ ] 对照表中每行的新文件夹格式为 `prds/[ID]-[标题]/`
- [ ] 输出包含 registry 路径更新说明
- [ ] 输出包含 drafts/ 孤立目录清理清单（或明确说明无孤立目录）
- [ ] 输出包含验收检查步骤（至少包含一条可运行的命令示例）
- [ ] AI **未**自动重命名或删除任何文件（仅输出方案）

## TC-MIG-02 已符合新格式的文件夹正确标注

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **测试目标** | 验证 prds/ 中已带 ID 前缀的文件夹在迁移清单中被标注为无需迁移 |
| **前置条件** | `prds/` 中有一个已符合新格式的文件夹（如 `prds/F-009-某功能/`），同时有未迁移的旧格式文件夹 |

**测试输入**
```
执行存量迁移
```

**预期行为**
1. AI 识别出已符合新格式的文件夹
2. 在迁移清单中标注"✅ 已符合新格式，无需迁移"
3. 仅列出需要迁移的旧格式文件夹

**检查要点**
- [ ] 已符合新格式的文件夹标注"✅ 已符合新格式"或"无需迁移"
- [ ] 未将已符合新格式的文件夹列入待迁移列表
- [ ] registry 中不存在但 prds/ 目录存在的情况有"⚠️ 注意"标注
