# 测试用例：/new-prd 命令

> 关联命令：`.claude/commands/new-prd.md`
> 前置条件：`prds/_registry.md` 存在且已初始化

---

## TC-NP-01 正常创建 story-card

| 字段 | 内容 |
|------|------|
| **状态** | ✅ 2026-04-08 |
| **测试目标** | 验证基础创建流程：ID 分配、文件结构、registry 更新 |
| **前置条件** | registry 中当前无 SC 类型 PRD（或最大编号已知） |

**测试输入**
```
/new-prd story-card 用户登录失败提示优化
```

**预期行为**
1. 读取 `prds/_registry.md`，找到当前 SC 最大编号
2. 分配新 ID（如 SC-001）
3. 创建 `prds/用户登录失败提示优化/` 文件夹
4. 从 `templates/story-card.md` 复制，替换占位符（ID、标题、日期）
5. 创建 `CHANGELOG.md`（含初始版本记录）
6. 创建空 `archive/` 文件夹
7. 在 `_registry.md` 追加一行
8. 因无额外需求描述，保留空模板并告知用户下一步

**检查要点**
- [ ] 文件路径：`prds/用户登录失败提示优化/prd.md` 存在
- [ ] prd.md 中 `id` 字段为 SC-001（或递增后正确值），非 SC-XXX
- [ ] prd.md 中 `created` 和 `updated` 为今天日期
- [ ] `_registry.md` 活跃 PRD 表格中增加了对应行
- [ ] AI 输出中告知了用户可通过哪些方式继续填写内容

## TC-NP-02 创建 feature PRD（含需求描述，自动填充）

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **测试目标** | 验证有上下文时 AI 自动填充内容，并生成 fields.md |
| **前置条件** | registry 中当前无 F 类型 PRD |

**测试输入**
```
/new-prd feature 消息推送设置

需求背景：用户希望能自主控制接收哪些类型的系统消息推送（订单状态、营销活动、系统公告），并可分渠道设置（App 推送、邮件、短信）。管理员可在后台配置哪些类型强制开启不可关闭。
```

**预期行为**
1. 分配 F-001（或递增后正确值）
2. 创建完整文件夹结构（含 fields.md）
3. 基于需求描述自动填写 prd.md 各章节
4. 生成 fields.md 并填入可推断的核心字段
5. CHANGELOG 注明"由 AI 基于用户描述生成"

**检查要点**
- [ ] `fields.md` 文件存在且有内容（非空占位符）
- [ ] prd.md 中用户故事符合 As a / I want to / So that 格式
- [ ] prd.md 中至少包含用户故事、功能概述、验收标准三个章节
- [ ] 验收标准至少 3 条且可测试
- [ ] CHANGELOG.md 注明内容来源

## TC-NP-03 缺少类型参数时的处理

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **测试目标** | 验证缺少类型时 AI 主动询问，不自行假设 |
| **前置条件** | 无特殊前置条件 |

**测试输入**
```
/new-prd 搜索结果排序优化
```

**预期行为**
- AI 识别到缺少类型参数
- 主动询问类型（story-card / feature / epic）
- **不**自行推断或创建任何文件

**检查要点**
- [ ] AI 输出中包含对类型的询问
- [ ] 未创建任何文件或文件夹
- [ ] 询问方式清晰，提供了三种类型的简要说明

## TC-NP-04 同名文件夹已存在时的处理

| 字段 | 内容 |
|------|------|
| **状态** | ✅ 2026-04-08（修复后） |
| **测试目标** | 验证存在同名文件夹时 AI 不覆盖，提示用户 |
| **前置条件** | `prds/用户登录失败提示优化/` 文件夹已存在（执行 TC-NP-01 后） |

**测试输入**
```
/new-prd story-card 用户登录失败提示优化
```

**预期行为**
- AI 检测到同名文件夹已存在
- 提示用户该路径已存在，说明已有 PRD 的 ID 和状态
- 询问用户意图：是要更新现有 PRD（用 `/update-prd`）还是新建不同名称的 PRD
- **不**覆盖现有文件

**检查要点**
- [ ] 现有 `prd.md` 未被修改（内容与执行前相同）
- [ ] AI 输出中提示了冲突并给出了两个选项
- [ ] AI 建议使用 `/update-prd` 处理更新场景
