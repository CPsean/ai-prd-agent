# 测试用例：/new-prd 命令

> 关联命令：`.claude/commands/new-prd.md`
> 前置条件：`prds/_registry.md` 存在且已初始化

---

## TC-NP-01 正常创建 story-card（草稿区）

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **测试目标** | 验证基础创建流程：文件创建在 drafts/，registry 暂不更新，结束时询问是否发布 |
| **前置条件** | registry 中当前无 SC 类型 PRD（或最大编号已知） |

**测试输入**
```
/new-prd story-card 用户登录失败提示优化
```

**预期行为**
1. 读取 `prds/_registry.md`，找到当前 SC 最大编号
2. 分配新 ID（如 SC-001）
3. 创建 `drafts/用户登录失败提示优化/` 文件夹
4. 从 `templates/story-card.md` 复制，替换占位符（ID、标题、日期）
5. 创建 `CHANGELOG.md`（含初始版本记录）
6. 创建空 `archive/` 文件夹
7. **不**更新 `_registry.md`（草稿阶段不注册）
8. 因无额外需求描述，保留空模板并告知用户下一步
9. 主动询问：「是否立即移入正式区（prds/）？A 留草稿 / B 移入正式区」

**检查要点**
- [ ] 文件路径：`drafts/用户登录失败提示优化/prd.md` 存在
- [ ] `prds/用户登录失败提示优化/` **不存在**
- [ ] prd.md 中 `id` 字段为 SC-001（或递增后正确值），非 SC-XXX
- [ ] prd.md 中 `created` 和 `updated` 为今天日期
- [ ] `_registry.md` 活跃 PRD 表格中**未增加**对应行
- [ ] AI 输出末尾包含 A/B 发布询问

## TC-NP-02 创建 feature PRD（含需求描述，自动填充）

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **测试目标** | 验证有上下文时 AI 自动填充内容，按需生成 fields.md，结束后询问是否发布 |
| **前置条件** | registry 中当前无 F 类型 PRD |

**测试输入**
```
/new-prd feature 消息推送设置

需求背景：用户希望能自主控制接收哪些类型的系统消息推送（订单状态、营销活动、系统公告），并可分渠道设置（App 推送、邮件、短信）。管理员可在后台配置哪些类型强制开启不可关闭。
```

**预期行为**
1. 分配 F-001（或递增后正确值）
2. 在 `drafts/消息推送设置/` 创建完整文件夹结构
3. 基于需求描述自动填写 prd.md 各章节
4. 判断需要 fields.md（需求涉及数据字段），创建并填入核心字段
5. CHANGELOG 注明"由 AI 基于用户描述生成"
6. **不**更新 `_registry.md`
7. 主动询问是否移入正式区

**检查要点**
- [ ] 文件路径：`drafts/消息推送设置/prd.md` 存在（非 `prds/`）
- [ ] `fields.md` 文件存在且有内容（非空占位符）
- [ ] prd.md 中用户故事符合 As a / I want to / So that 格式
- [ ] prd.md 中至少包含用户故事、功能概述、验收标准三个章节
- [ ] 验收标准至少 3 条且可测试
- [ ] CHANGELOG.md 注明内容来源
- [ ] `_registry.md` **未增加**对应行
- [ ] AI 输出末尾包含 A/B 发布询问

## TC-NP-04 同名文件夹已存在时的处理

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **测试目标** | 验证 drafts/ 和 prds/ 任一存在同名文件夹时 AI 不覆盖，提示用户 |
| **前置条件** | `drafts/用户登录失败提示优化/` 已存在（执行 TC-NP-01 后）或 `prds/用户登录失败提示优化/` 已存在 |

**测试输入**
```
/new-prd story-card 用户登录失败提示优化
```

**预期行为**
- AI 检测到同名文件夹在 drafts/ 或 prds/ 中已存在
- 告知用户该路径已存在（说明在哪个区域、ID 和当前状态）
- 给出两个选项：更新现有 PRD（`/update-prd`）或新建不同名称的 PRD
- **不**覆盖现有文件

**检查要点**
- [ ] 现有 `prd.md` 未被修改
- [ ] AI 明确说明是在 drafts/ 还是 prds/ 中发现冲突
- [ ] AI 建议使用 `/update-prd [标题] [变更描述]` 处理更新场景

## TC-NP-05 用户选择 B 后草稿移入正式区

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **测试目标** | 验证确认发布后：文件移入 prds/，registry 注册，drafts/ 原文件夹删除 |
| **前置条件** | TC-NP-01 执行后，`drafts/用户登录失败提示优化/` 存在 |

**测试输入**
```
B
```
（接 TC-NP-01 的 A/B 询问）

**预期行为**
1. 将 `drafts/用户登录失败提示优化/` 整体移动到 `prds/用户登录失败提示优化/`
2. 在 `prds/_registry.md` 追加注册行
3. 告知用户 PRD 已注册，可运行 `/prd-summary` 开始评审

**检查要点**
- [ ] `prds/用户登录失败提示优化/prd.md` 存在
- [ ] `drafts/用户登录失败提示优化/` **不再存在**
- [ ] `_registry.md` 活跃 PRD 表格中增加了对应行
- [ ] AI 提示可运行 `/prd-summary 用户登录失败提示优化`
