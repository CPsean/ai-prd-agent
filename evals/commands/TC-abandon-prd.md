# 测试用例：/abandon-prd 命令

> 关联命令：`.claude/commands/abandon-prd.md`
> 关联 PRD：F-009 PRD生命周期一致性（PRD-GEN-010）
> 前置条件：`drafts/_draft-registry.md` 存在

---

## TC-AP-01 正常放弃草稿

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **测试目标** | 验证正常流程：二次确认后删除草稿目录并清理预注册条目 |
| **前置条件** | `drafts/F-009-某功能/` 存在（含 prd.md、CHANGELOG.md、archive/），`drafts/_draft-registry.md` 中有 F-009 对应条目 |

**测试输入**
```
/abandon-prd 某功能
```
然后在二次确认时回复：
```
确认
```

**预期行为**
1. AI 查找 `drafts/_draft-registry.md` 中与"某功能"匹配的条目，获取 ID（F-009）
2. 输出二次确认提示，包含标题和 ID
3. 用户回复"确认"后，删除 `drafts/F-009-某功能/` 整个目录
4. 从 `drafts/_draft-registry.md` 删除 F-009 条目
5. 输出完成提示："草稿「某功能」已放弃，目录和预注册 ID 已清理。"

**检查要点**
- [ ] AI 输出了二次确认提示，包含标题和 ID
- [ ] `drafts/F-009-某功能/` 目录不存在（已删除）
- [ ] `drafts/_draft-registry.md` 中无 F-009 行
- [ ] AI 输出包含"已放弃"或"已清理"等确认字样

| 字段 | 内容 |
|------|------|

## TC-AP-04 预注册条目不存在但目录存在

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **测试目标** | 验证异常状态下（目录存在但预注册条目缺失），AI 仍能完成目录删除并提示异常 |
| **前置条件** | `drafts/F-009-某功能/` 目录存在，但 `drafts/_draft-registry.md` 中**无** F-009 条目（手动删除了条目） |

**测试输入**
```
/abandon-prd 某功能
```
然后在二次确认时回复：
```
确认
```

**预期行为**
1. AI 在 `drafts/_draft-registry.md` 中未找到匹配条目，但通过目录搜索在 `drafts/` 中找到 `F-009-某功能/`
2. 输出二次确认（包含异常状态说明）
3. 用户确认后删除目录
4. 输出提示："预注册条目不存在，目录已手动清理"

**检查要点**
- [ ] `drafts/F-009-某功能/` 目录已删除
- [ ] AI 输出中提及"预注册条目不存在"或类似异常说明
- [ ] 操作正常完成，未因异常状态而中断
