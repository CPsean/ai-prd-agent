# 测试用例：/generate-page-spec 命令

> 关联命令：`.claude/commands/generate-page-spec.md`
> 参数格式：`/generate-page-spec [标题或ID]`
> 前置条件：目标 PRD 已在 `prds/` 正式区注册

---

## TC-GPS-01 正常生成页面规格卡（Feature PRD）

| 字段 | 内容 |
|------|------|
| **状态** | ✅ 2026-04-10 |
| **测试目标** | 验证基础流程：识别页面范围 → 输出清单 → 用户确认 → 生成页面规格卡 → 保存至正确路径 |
| **前置条件** | `prds/消息推送设置/prd.md` 存在，包含"页面 & 交互说明"章节，定义了至少 2 个页面/弹窗 |

**测试输入**
```
/generate-page-spec 消息推送设置
```

**预期行为**
1. 从 `_registry.md` 定位 `prds/消息推送设置/prd.md`，读取内容
2. 识别"页面 & 交互说明"章节中的所有页面对象（页面、弹窗、抽屉）
3. 输出页面清单并等待用户确认：列出每个页面名称 + 一句话说明
4. 用户确认后，逐页生成规格卡（区块结构 / 状态定义 / 操作说明 / 关键文案）
5. 在规格卡顶部插入元数据头（`source-prd-version` / `last-updated`）
6. 保存至 `prds/消息推送设置/page-spec.md`
7. 更新 `prd.md` frontmatter：新增 `has-page-spec: true` 和 `page-spec-path: page-spec.md`
8. 输出确认信息：文件路径、覆盖页面数、引导运行 `/generate-prototype`

**检查要点**
- [ ] `prds/消息推送设置/page-spec.md` 文件存在
- [ ] page-spec.md 顶部有 `source-prd-version` 元数据，值与 `prd.md` 当前 version 一致
- [ ] page-spec.md 顶部有 `last-updated: 今天日期`
- [ ] 每个识别到的页面对象均有：区块结构表格、状态定义表格、操作说明表格、关键文案表格
- [ ] `prd.md` frontmatter 中 `has-page-spec` 已变为 `true`
- [ ] `prd.md` frontmatter 中出现 `page-spec-path: page-spec.md`
- [ ] AI 输出第一轮为页面清单确认，**不**直接生成规格内容
- [ ] 确认后生成的规格中页面数量与 PRD 识别页面数一致
