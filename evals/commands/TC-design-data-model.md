# 测试用例：/design-data-model 命令

> 关联命令：`.claude/commands/design-data-model.md`
> 参数格式：`/design-data-model [业务场景]`
> 前置条件：无特殊前置文件
> Codex 等效输入：「设计数据模型：[场景]」或「帮我设计[场景]的数据库表结构」

---

## TC-DM-01 正常数据模型设计

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **Codex 状态** | — |
| **测试目标** | 验证输出包含 ER 图、Schema 表格和设计说明，符合企业级标准 |
| **前置条件** | 无特殊前置条件 |

**测试输入**
```
/design-data-model 电商订单系统，包含：用户下单、订单详情（多商品）、支付记录、物流跟踪
```

**预期行为**
1. 领域分析：识别聚合根（订单）和实体（订单详情、支付记录、物流记录）
2. 定义实体间关系（订单 1:N 订单详情、订单 1:1 支付、订单 1:N 物流）
3. 输出 ER 图（Mermaid 语法）
4. 输出每个实体的 Schema 表格
5. 输出设计说明（聚合根选择理由、索引策略）

**检查要点**
- [ ] 包含 Mermaid ER 图（```mermaid erDiagram 格式）
- [ ] Schema 表格格式正确（字段名/类型/长度/必填/默认值/索引/说明）
- [ ] 主键使用雪花 ID（BigInt）
- [ ] 包含审计字段：`created_at`、`updated_at`、`created_by`、`updated_by`
- [ ] 包含软删除字段：`is_deleted`
- [ ] 包含乐观锁字段：`version`
- [ ] 金额字段使用 `DECIMAL`，未使用 `FLOAT` 或 `DOUBLE`
- [ ] 有索引策略说明

## TC-DM-02 业务场景模糊时停止并提问

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **Codex 状态** | — |
| **测试目标** | 验证过于模糊的输入触发提问，不凭空建模 |
| **前置条件** | 无特殊前置条件 |

**测试输入**
```
/design-data-model 做个管理系统
```

**预期行为**
- AI 识别到无法确定具体业务实体
- 停止并提问："请提供具体的业务实体或核心字段，以便继续建模。"
- **不**自行假设是什么管理系统

**检查要点**
- [ ] AI 未输出任何 Schema 或 ER 图
- [ ] 提问内容要求用户提供具体业务实体
- [ ] 未自行假设管理对象（如不自行决定是"用户管理"还是"库存管理"）

## TC-DM-03 类型规范验证（金额字段）

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **Codex 状态** | — |
| **测试目标** | 专项验证金额/财务相关字段的类型合规性 |
| **前置条件** | 无特殊前置条件 |

**测试输入**
```
/design-data-model 充值与消费记录，包含：用户充值余额、消费流水、退款记录，金额精确到分
```

**预期行为**
1. 所有金额字段使用 `DECIMAL(p,s)` 类型
2. 不使用 `FLOAT` 或 `DOUBLE`
3. 精度设计合理（如 `DECIMAL(12,2)` 表示最大 10 亿，精确到分）

**检查要点**
- [ ] 充值金额字段类型为 `DECIMAL`
- [ ] 消费金额字段类型为 `DECIMAL`
- [ ] 退款金额字段类型为 `DECIMAL`
- [ ] 未出现 `FLOAT` 或 `DOUBLE` 类型
- [ ] VARCHAR 使用了具体长度（如 `VARCHAR(64)`），非无限制的 `TEXT`

## TC-DM-04 N:M 关系的中间表设计

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **Codex 状态** | — |
| **测试目标** | 验证多对多关系时正确创建中间表 |
| **前置条件** | 无特殊前置条件 |

**测试输入**
```
/design-data-model 标签系统，文章可以有多个标签，一个标签可以关联多篇文章
```

**预期行为**
1. 识别文章与标签为 N:M 关系
2. 创建中间表（如 `article_tag`）
3. 中间表包含双外键和联合索引

**检查要点**
- [ ] ER 图正确表示 N:M 关系
- [ ] 存在中间表（article_tag 或类似命名）
- [ ] 中间表包含 `article_id` 和 `tag_id` 外键
- [ ] 中间表有联合索引或复合主键
