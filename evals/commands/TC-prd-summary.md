# 测试用例：/prd-summary 命令

> 关联命令：`.claude/commands/prd-summary.md`
> 前置条件：`prds/` 目录下存在至少一个已创建的 PRD
> Codex 等效输入：「给我[标题]的评审摘要」或「帮我总结[标题]的 PRD」

---

## TC-PS-01 正常 PRD 摘要输出

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **Codex 状态** | — |
| **测试目标** | 验证摘要输出结构与信息密度符合 skill-outputs 示例 |
| **前置条件** | 使用 examples/good-prd/ 下的示例 PRD（先通过 /new-prd 或手动创建一个 PRD） |

**测试输入**
```
/prd-summary 消息通知设置
```

**预期行为**
1. 从 `_registry.md` 定位 PRD 路径
2. 读取 `prd.md` 和 `fields.md`（如有）
3. 输出结构化摘要：基本信息、一句话描述、核心用户故事、关键业务规则、验收标准速览、已知边界
4. 摘要结束后**自动执行质检**，输出质检报告

**检查要点**
- [ ] 输出包含"一句话描述"段落
- [ ] 业务规则以表格形式列出（不是大段文字）
- [ ] 验收标准精简为测试重点（非完整复制）
- [ ] 摘要末尾包含质检报告（✅/⚠️/❌ 计数）
- [ ] 质检报告对照 `rules/prd-quality-gates.md` 中的检查项

---

## TC-PS-02 指定不存在的 PRD

| 字段 | 内容 |
|------|------|
| **状态** | ✅ 2026-04-08（修复后） |
| **Codex 状态** | — |
| **测试目标** | 验证 PRD 不存在时 AI 正确处理，不捏造内容 |
| **前置条件** | 无特殊前置条件 |

**测试输入**
```
/prd-summary 购物车结算流程重构
```

**预期行为**
1. 读取 `_registry.md`，在活跃 PRD 表格中搜索
2. 未找到匹配项时，明确告知用户该 PRD 不存在
3. 提示用户可以通过 `/new-prd` 创建，或确认标题是否拼写有误
4. **不**捏造或猜测 PRD 内容

**检查要点**
- [ ] AI 未输出任何虚构的 PRD 摘要内容
- [ ] 明确告知"未在注册表中找到该 PRD"
- [ ] 提供了后续操作建议（创建/确认标题）

## TC-PS-03 摘要后质检自动触发验证

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **Codex 状态** | — |
| **测试目标** | 专项验证：质检报告必须在每次 /prd-summary 末尾自动出现 |
| **前置条件** | 任意已存在的 PRD |

**测试输入**
```
/prd-summary [任意已存在的 PRD 标题]
```

**预期行为**
- 摘要内容输出后，紧接着输出质检报告
- 质检报告包含：通过项数、待完善项数、未通过项数
- 未通过项必须给出具体的规则编号和改写建议

**检查要点**
- [ ] 输出末尾存在"PRD 质检报告"标题
- [ ] 包含 ✅/⚠️/❌ 三类计数
- [ ] 每个 ❌ 项有对应的规则编号（如 G3-03）和修复建议
- [ ] 质检结论与 `rules/prd-quality-gates.md` 中的规则逻辑一致
