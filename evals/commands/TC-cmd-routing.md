# 测试用例：命令路由（F-004 命令管理）

> 关联 PRD：`prds/命令管理/prd.md`（F-004）
> 关联文件：`CLAUDE.md`（意图匹配规则）、`rules/routing-signals.md`（X-Y 风险信号词表）

## TC-CMD-01 单一匹配：直接执行不确认

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **功能编号** | CMD-RTG-002 |
| **测试目标** | 意图明确匹配单一命令时，AI 直接执行，不弹出确认 |

**测试输入**
```
帮我新建一个功能 PRD，标题叫「用户消息推送」
```

**预期行为**
1. AI 识别意图 → `/new-prd feature 用户消息推送`
2. 直接进入 new-prd 执行流程（询问新功能/迭代优化）
3. **不**先问「你是想用哪个命令吗？」或「确认执行 /new-prd 吗？」

**检查要点**
- [ ] AI 未询问确认，直接进入 new-prd 步骤 0
- [ ] 不展示命令选择菜单

## TC-CMD-05 模糊意图：追问 1 次区分

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **功能编号** | CMD-RTG-003 |
| **测试目标** | 意图匹配多个候选命令时，AI 追问 ≤1 个区分问题 |

**测试输入**
```
我想写一下审批流这个需求
```

**预期行为**
1. 识别可能匹配 `/requirement-clarifier` 或 `/new-prd`
2. 追问 1 个区分问题，格式：「你是想先澄清需求方向，还是需求已经明确、直接写 PRD？」
3. 等待 PM 回答后执行对应命令

**检查要点**
- [ ] 只追问了 1 个问题（不能问 2 个）
- [ ] 追问用业务语言，未暴露命令技术名称（如不说「/requirement-clarifier 还是 /new-prd」）
- [ ] PM 回答后直接执行，不再追问

## TC-CMD-09 路由纠错：有文件时询问删除

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **功能编号** | CMD-ERR-001 |
| **测试目标** | 命令执行中 PM 发出纠错信号，AI 终止并询问是否删除已产生文件 |

**测试输入**（多轮对话）
```
Round 1 - PM: 帮我起草一个需求
Round 1 - AI: [进入 requirement-clarifier，创建了 drafts/xxx/rdd.md]
Round 2 - PM: 不对，我是要直接写 PRD
```

**预期行为**
1. AI 识别「不对」为纠错信号
2. **立即终止** requirement-clarifier 流程
3. 检测到已产生 `drafts/xxx/rdd.md`
4. 询问：「已产生文件 drafts/xxx/rdd.md，是否删除？」
5. 等待 PM 确认后才删除，不自动删除

**检查要点**
- [ ] 当轮输出中止了 requirement-clarifier 流程
- [ ] 询问是否删除文件，未自动删除
- [ ] 提到了具体文件路径
- [ ] 等待 PM 明确回复「是/否」才继续
