# 测试用例：/update-prd 命令

> 关联命令：`.claude/commands/update-prd.md`
> 参数格式：`/update-prd [标题] [变更描述]`
> 前置条件：目标 PRD 存在于 `prds/` 或 `drafts/` 中
> Codex 等效输入：「更新[标题]的 PRD：[变更描述]」
> Codex 状态：已适配

---

## TC-UP-01 小变更（版本号 +0.1，正式区 PRD）

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **测试目标** | 验证小变更流程：归档旧版、更新日期、版本 +0.1、追加 CHANGELOG、更新 registry |
| **前置条件** | `prds/消息通知设置/prd.md` 版本为 v1.0，已在 registry 注册 |

**测试输入**
```
/update-prd 消息通知设置 根据评审意见，将 AC-1 中"误差不超过1次刷新周期"修改为"页面进入后2秒内（P95）完成数据同步"
```

**预期行为**
1. 从 `_registry.md` 定位 `prds/消息通知设置/prd.md`
2. 将当前版本归档到 `archive/prd-v1.0.md`
3. 修改 `prd.md` 中 AC-1 内容
4. 更新 frontmatter：`version: "1.1"`、`updated: 今天日期`
5. 在 `CHANGELOG.md` 追加 v1.1 变更记录
6. 更新 `_registry.md` 中对应行的 `updated` 列
7. **不**询问是否发布（已在正式区）

**检查要点**
- [ ] `archive/prd-v1.0.md` 存在且内容为旧版本
- [ ] `prd.md` 中 version 为 "1.1"
- [ ] `prd.md` 中 updated 为今天日期
- [ ] `CHANGELOG.md` 中存在 "v1.1" 章节，记录了本次变更
- [ ] AC-1 内容已更新，不再包含"刷新周期"模糊描述
- [ ] 未出现"是否发布"询问

| 字段 | 内容 |

## TC-UP-03 更新草稿区 PRD，完成后询问是否发布

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **测试目标** | 验证草稿区 PRD 可被正常更新，且更新后主动询问是否移入正式区 |
| **前置条件** | `drafts/用户登录失败提示优化/prd.md` 存在（执行 TC-NP-01 后），未在 registry 注册 |

**测试输入**
```
/update-prd 用户登录失败提示优化 补充 AC-2：连续失败 5 次后账号锁定 30 分钟，锁定期间提示用户剩余解锁时间
```

**预期行为**
1. registry 中未找到该标题
2. 在 `drafts/用户登录失败提示优化/` 找到 prd.md
3. 执行归档 + 内容更新 + 版本 +0.1 + CHANGELOG
4. **不**更新 registry（仍在草稿区）
5. 更新完成后主动询问：「草稿已更新。是否现在移入正式区？A 继续完善 / B 正式发布」

**检查要点**
- [ ] `drafts/用户登录失败提示优化/archive/prd-v1.0.md` 存在
- [ ] `drafts/用户登录失败提示优化/prd.md` version 为 "1.1"
- [ ] CHANGELOG 有 v1.1 记录
- [ ] `_registry.md` **未变化**
- [ ] AI 输出末尾包含 A/B 发布询问

## TC-UP-05 存量模板指引自动清理（F-012 PRD-UPD-010）

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **测试目标** | 验证 `/update-prd` 在执行变更后自动扫描并删除残留的模板指引文字，并在输出中报告清理数量和摘要 |
| **前置条件** | `prds/消息通知设置/prd.md` 中残留了 `> **编写规则**：` 和 `> 端清单参考：` 两处模板指引文字（旧版遗留）；PRD 已在正式区 |

**测试输入**
```
/update-prd 消息通知设置 补充 AC-3 关于通知频率限制的说明
```

**预期行为**
1. 步骤 3 执行变更（补充 AC-3）
2. 步骤 3.5 扫描全文，匹配到 2 处模板指引（`> **编写规则**：` 及其续行、`> 端清单参考：`）
3. 静默删除这 2 处，清理多余空行
4. 步骤 7 输出确认中追加：「已清理 2 处模板残留文字：[编写规则... / 端清单参考...]」

**检查要点**
- [ ] `prd.md` 中不含 `> **编写规则**：` 或 `> 端清单参考：` 字样
- [ ] 版本号正常升级（+0.1）
- [ ] AI 输出末尾包含"已清理 N 处模板残留文字"及内容摘要
- [ ] 变更的 AC-3 内容已正确写入 `prd.md`（变更本身未受影响）
