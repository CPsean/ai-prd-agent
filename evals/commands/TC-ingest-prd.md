# 测试用例：/ingest-prd 命令

> 关联命令：`.claude/commands/ingest-prd.md`、`.claude/skills/ingest-prd.md`
> 前置条件：`prds/_registry.md` 存在；`context/` 相关文件已初始化（如有）
> Codex 等效输入：「录入历史PRD」或「帮我归档已有的需求文档」

---

## TC-IP-01 提供需求 ID 的历史 PRD 录入

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **Codex 状态** | — |
| **测试目标** | 验证 Step 0.5 需求 ID 确认流程：找到匹配后继续，不跳过确认 |
| **前置条件** | `context/iteration-requirement-list.md` 中存在 REQ-101 对应的需求记录 |

**测试输入**
```
/ingest-prd

需求ID：REQ-101

以下是原始 PRD 内容：

---
# 用户头像上传功能

## 功能说明
用户可以在个人资料页上传自定义头像，支持 JPG/PNG 格式，文件大小不超过 2MB。上传后系统自动裁剪为正方形并压缩。

## 验收标准
- 用户可上传 JPG/PNG 格式图片
- 文件超过 2MB 时提示错误
- 上传成功后头像立即更新
---
```

**预期行为**
1. Step 0：读取 `_registry.md` 和 `iteration-requirement-list.md`
2. Step 0.5：在清单中查找 REQ-101，确认标题匹配后输出确认语
3. Step 1：判断层级（单功能，可能为 story-card 或 feature）
4. Step 2~7：创建文件、填充内容、更新 registry 和清单
5. Step 8：输出录入报告，标注待补全项（验收标准只有 3 条，边界条件缺失）

**检查要点**
- [ ] Step 0.5 有明确的确认输出，格式：`确认：需求 REQ-101「xxx」与本PRD内容匹配，继续执行`
- [ ] 创建了正确的文件夹和 prd.md
- [ ] 录入报告中标注了"边界条件缺失"为待补全项
- [ ] `iteration-requirement-list.md` 中 REQ-101 行的"关联PRD"列已更新

## TC-IP-02 不提供需求 ID（从清单中匹配）

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **Codex 状态** | — |
| **测试目标** | 验证未提供 ID 时 AI 列出候选项请用户确认，而非自行匹配写入 |
| **前置条件** | `context/iteration-requirement-list.md` 中存在多条与"头像"相关的需求 |

**测试输入**
```
/ingest-prd

以下是原始 PRD 内容（未知需求ID）：

---
# 用户头像上传功能
...（同 TC-IP-01）
---
```

**预期行为**
1. Step 0.5 情况B：从清单按关键词"头像"搜索，找到 1-3 个候选
2. 列出候选项请用户确认（格式：`· [ID]「标题」— [所属迭代]`）
3. **等待用户回复**，不自行写入任何文件

**检查要点**
- [ ] AI 列出候选需求 ID，格式符合规范
- [ ] AI **未**创建任何文件（等待用户确认）
- [ ] 候选项数量 ≤ 3
- [ ] 如果清单中完全无匹配，标注"未匹配"并询问是否继续
