# 测试用例：/prd-qa 命令

> 关联命令：`.claude/commands/prd-qa.md`
> 参数格式：`/prd-qa [问题]`
> 前置条件：`prds/` 目录下存在已注册的正式 PRD
> Codex 等效输入：直接提问，如「[标题] 这个功能中，[具体问题]？」（Codex 根据 PRD 回答）

---

## TC-PQ-01 正常问答（答案在 PRD 中明确定义）

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **测试目标** | 验证 AI 能精准定位并直接回答 PRD 中有明确定义的问题 |
| **前置条件** | `prds/消息通知设置/prd.md` 存在且包含完整验收标准 |

**测试输入**
```
/prd-qa 消息通知设置的验收标准第一条是什么？
```

**预期行为**
1. 从 `_registry.md` 定位 PRD
2. 读取 `prd.md`，找到验收标准章节
3. 直接给出 AC-1 的内容，注明来源章节
4. 不大段复述 PRD 原文

**检查要点**
- [ ] 答案准确，与 `prd.md` 中 AC-1 内容一致
- [ ] 注明了来源章节（如"来自验收标准 AC-1"）
- [ ] 回答简洁，未大段复制 PRD 原文

## TC-PQ-02 问题涉及 PRD 中未定义的内容

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **Codex 状态** | ✅ 2026-04-13 |
| **测试目标** | 验证 PRD 中未定义的内容不胡编，并主动提示缺陷 |
| **前置条件** | 目标 PRD 中未定义 Session 超时时间 |

**测试输入**
```
/prd-qa 消息通知设置的 Session 超时时间是多少？
```

**预期行为**
1. 定位并读取 PRD
2. 搜索后发现 PRD 中未定义 Session 超时时间
3. 明确说明"PRD 中未定义此项"
4. 输出 ⚠️ 缺陷提示，建议运行 `/update-prd` 补充

**检查要点**
- [ ] AI 未捏造任何超时时间数值
- [ ] 明确说明"PRD 中未定义此项"
- [ ] 包含 ⚠️ 缺陷提示
- [ ] 建议使用 `/update-prd` 补充

## TC-PQ-03 通过 ID 定位 PRD

| 字段 | 内容 |

**检查要���**

- [ ] 回答内容与实际文件状态一致

## TC-PQ-04 询问草稿区 PRD 被拒绝

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **测试目标** | 验证草稿区 PRD 不可被 /prd-qa 读取 |
| **前置条件** | `drafts/用户登录失败提示优化/prd.md` 存在，`prds/` 中无此 PRD |

**测试输入**
```
/prd-qa 用户登录失败提示优化的验收标准是什么？
```

**预期行为**
- 查 `_registry.md` 未找到该 PRD
- 告知用户"该 PRD 尚未移入正式区，请先确认发布后再使用 `/prd-qa`"
- **不**读取 drafts/ 中的内容回答

**检查要点**
- [ ] AI 未读取 drafts/ 中的草稿内容
- [ ] 明确说明"尚未移入正式区"
- [ ] 提供了发布后再使用的引导

## TC-PQ-05 未指定目标 PRD 时询问

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **测试目标** | 验证未指定目标时 AI 主动询问，不猜测 |
| **前置条件** | 无特殊前置条件 |

**测试输入**
```
/prd-qa 这个功能的边界条件有哪些？
```

**预期行为**
- AI 识别到未指定具体 PRD
- 主动询问"请问是关于哪个功能的问题？"
- **不**随意选择一个 PRD 回答

**检查要点**
- [ ] AI 未自行选择 PRD 回答
- [ ] 输出中包含对目标 PRD 的询问
- [ ] 未创建任何文件
