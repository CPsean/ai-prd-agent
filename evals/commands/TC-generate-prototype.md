# 测试用例：/generate-prototype 命令（F-005 原型生成）

> 关联 PRD：`prds/原型生成/prd.md`（F-005）
> 关联命令：`.claude/commands/generate-prototype.md`
> 参数格式：`/generate-prototype [标题或ID]`
> 前置条件：目标 PRD 已在 `prds/` 正式区注册
> Codex 等效输入：「生成原型：[标题]」或「帮我做[标题]的交互原型」

---

## TC-GP-01 正常生成原型（有 page-spec + 有截图库）

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **Codex 状态** | — |
| **功能编号** | PROTO-PRE-001, PROTO-STY-001, PROTO-GEN-003, PROTO-LNK-001 |
| **测试目标** | 验证完整路径：规格卡存在 + 截图库存在时，生成含视觉还原的 HTML 原型 |
| **前置条件** | `prds/消息推送设置/page-spec.md` 存在，`context/screenshots/消息推送/` 下有截图，PRD 定义了至少 2 个页面 |

**测试输入**
```
/generate-prototype 消息推送设置
```

**预期行为**
1. 检查 `prds/消息推送设置/page-spec.md` 存在 → 读取
2. 读取 `context/screenshots/消息推送/` 截图，提取视觉风格
3. 输出规划清单（页面名称、元素、交互规则），等待用户确认
4. 确认后按规格卡页面列表逐页生成 HTML 文件
5. 生成 index.html 作为原型入口，含页面导航
6. 输出至 `outputs/prototypes/消息推送设置/`
7. 创建 `outputs/prototypes/消息推送设置/prototype-meta.md`
8. 更新 `prd.md` frontmatter：`has-prototype: true`
9. 输出确认信息：文件路径、页面清单、预览提示

**检查要点**
- [ ] 原型输出在 `outputs/prototypes/消息推送设置/` 目录
- [ ] 有 index.html 入口文件 + 各页面独立 HTML 文件
- [ ] HTML 为纯 HTML + CSS + JS，无外部 CDN 依赖
- [ ] 原型视觉风格体现截图中的产品样式（颜色、字体、布局）
- [ ] 页面间有跳转链接（非 Tab 切换）
- [ ] 表单元素可交互（输入框、下拉、按钮）
- [ ] `prototype-meta.md` 已创建，含 prd-version 和 generated-at
- [ ] `prd.md` 中 `has-prototype` 已改为 `true`
