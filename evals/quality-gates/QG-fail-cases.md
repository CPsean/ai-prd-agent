# 质检失败用例集

> 每条用例包含一段故意有缺陷的 PRD 片段，执行质检时应触发对应的质检项。
>
> **执行方式**：将下方 prompt 粘贴到新对话，AI 应识别出对应的 ❌ 或 ⚠️ 项。

---

## QG-F-01 缺少用户故事章节（触发 G2-01）

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **预期质检结果** | G2-01 ❌ 必须 |

**测试 Prompt**
```
请对以下 PRD 片段执行质检，对照 rules/prd-quality-gates.md 输出质检报告：

---
type: story-card
id: SC-TEST-01
title: 订单搜索功能
status: draft
version: "1.0"
created: 2026-04-07
updated: 2026-04-07
---

## 功能概述

用户可以在订单列表页通过关键词搜索订单，支持按订单号、收货人姓名进行搜索。

## 验收标准

- [ ] AC-1：输入订单号后，列表实时过滤展示匹配结果
- [ ] AC-2：无匹配结果时展示"暂无相关订单"
- [ ] AC-3：搜索框支持清空操作，清空后恢复完整列表

## 边界条件 & 异常处理

| 场景 | 处理方式 |
|------|----------|
| 输入特殊字符 | 自动过滤，不报错 |
| 搜索词长度超过50字 | 截断至50字后执行搜索 |
| 网络异常 | 提示"搜索失败，请重试" |
```

**检查要点**
- [ ] 质检报告中 G2-01 标记为 ❌
- [ ] 质检报告明确指出缺少"用户故事"章节
- [ ] 给出补充用户故事的格式建议

## QG-F-05 缺少边界条件章节（触发 G4-01）

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **预期质检结果** | G4-01 ❌ 必须 |

**测试 Prompt**
```
请对以下 PRD 片段执行质检：

## 用户故事

> **As a** 用户，**I want to** 修改我的账户密码，**so that** 我可以保护账户安全。

## 功能概述

用户可在"账户安全"页面修改登录密码，需验证旧密码后方可设置新密码。

## 验收标准

- [ ] AC-1：输入正确旧密码后可成功设置新密码
- [ ] AC-2：新密码长度 8-20 位，必须含字母和数字
```

**检查要点**
- [ ] 质检报告中 G4-01 标记为 ❌
- [ ] 指出缺少"边界条件 & 异常处理"章节
- [ ] 列举应该补充的场景（旧密码错误/新旧密码相同/多次失败锁定等）

## QG-F-06 Feature PRD 无字段清单（触发 G5-01）

| 字段 | 内容 |
|------|------|
| **状态** | — |
| **预期质检结果** | G5-01 ❌ 必须 |

**测试 Prompt**
```
请对以下 Feature PRD 执行质检（注意：type 为 feature-prd）：

---
type: feature-prd
id: F-TEST-01
title: 用户积分兑换
status: in-review
version: "1.0"
created: 2026-04-07
updated: 2026-04-07
---

## 用户故事

> **As a** 会员用户，**I want to** 用积分兑换优惠券，**so that** 我可以在下次购物时节省费用。

## 验收标准

- [ ] AC-1：积分余额充足时可成功发起兑换
- [ ] AC-2：兑换成功后积分实时扣减，优惠券出现在"我的优惠券"
- [ ] AC-3：积分不足时按钮置灰并展示"积分不足"提示

## 边界条件 & 异常处理

| 场景 | 处理方式 |
|------|----------|
| 兑换接口超时 | 提示"兑换失败，请重试"，积分不扣减 |
| 优惠券库存为零 | 提示"该奖品已兑完" |
| 并发重复兑换 | 服务端幂等处理，只成功一次 |

## 字段清单

（暂无）
```

**检查要点**
- [ ] 质检报告中 G5-01 标记为 ❌
- [ ] 明确指出 feature-prd 类型必须有字段清单
- [ ] 从 PRD 内容推断应该包含哪些字段（如 points_required, coupon_id 等）
